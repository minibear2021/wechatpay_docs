分页查询投放计划列表




## 接口说明

支持商户：【普通服务商】

请求方式：【GET】 `/v3/marketing/partner/delivery-plan/delivery-plans/{brand_id}/delivery-plans`

请求域名：【主域名】https://api.mch.weixin.qq.com 使用该域名将访问就近的接入点

　　　　　【备域名】https://api2.mch.weixin.qq.com 使用该域名将访问异地的接入点，指引[点击查看](https://pay.weixin.qq.com/doc/v3/merchant/4012075113.md)

接口限频：100/秒（商户号维度）

## 请求参数

### Header 参数
| 参数名 | 类型 | 必填 | 描述 |
| --- |  --- | --- | --- |
| Authorization | string | true | 请参考 [签名认证](https://pay.weixin.qq.com/doc/v3/merchant/4012365342.md#%E8%AF%B7%E6%B1%82%E6%8E%A5%E5%8F%A3%E5%90%8E%E7%AB%AF%E7%AD%BE%E5%90%8D) 生成认证信息 |
| Accept | string | true | 请设置为 `application/json` |
### Path 参数
| 参数名 | 类型 | 必填 | 描述 |
| --- |  --- | --- | --- |
| brand_id | string | true | 【品牌ID】 创建投放计划的品牌ID，可登录 [合作伙伴平台](https://pay.weixin.qq.com/index.php/partner/public/home) ，在“品牌经营中心-合作品牌管理”页面查看已授权的brand\_id |
### Query 参数
| 参数名 | 类型 | 必填 | 描述 |
| --- |  --- | --- | --- |
| page_size | integer | false | 【分页大小】 批量查询下每页的数量，最大不超过50。不填默认为10。 |
| offset | integer | false | 【offset】 该次请求资源的起始位置。第一页从0开始 |
| plan_state | string | false | 【投放计划状态】 不填写表示查询所有状态的投放计划，填写表示查询特定状态的投放计划。已终止、已过期是终态。<br>可选取值<br><ul><li>`CREATED`:  创建成功</li><li>`TERMINATED`:  已终止</li><li>`EXPIRED`:  已过期</li><li>`DELIVERING`:  投放中</li><li>`PAUSED`:  已暂停</li></ul> |
| audit_state | string | false | 【投放计划审核状态】 不填写表示查询所有审核状态的投放计划，填写则表示查询特定状态的投放计划。<br>可选取值<br><ul><li>`AUDIT_INITIAL`:  初始化</li><li>`AUDIT_PROCESSING`:  审批中</li><li>`AUDIT_PASSED`:  审批通过</li><li>`AUDIT_REJECTED`:  审批拒绝</li></ul> |
| plan_id | string | false | 【投放计划ID】 投放计划ID，如果填入该字段，则只返回该ID对应的投放计划 |

| plan_list.delivery_start_time | string | true | 【投放开始时间】 投放开始时间，遵循 [rfc3339](https://datatracker.ietf.org/doc/html/rfc3339) 标准格式，格式为YYYY-MM-DDTHH:mm:ss+TIMEZONE，YYYY-MM-DD表示年月日，T出现在字符串中，表示time元素的开头，HH:mm:ss表示时分秒，TIMEZONE表示时区（+08:00表示东八区时间，领先UTC 8小时，即北京时间）。例如：2015-05-20T13:29:35+08:00表示，北京时间2015年5月20日 13点29分35秒。 |
| plan_list.delivery_end_time | string | true | 【投放结束时间】 投放结束时间，遵循 [rfc3339](https://datatracker.ietf.org/doc/html/rfc3339) 标准格式，格式为YYYY-MM-DDTHH:mm:ss+TIMEZONE，YYYY-MM-DD表示年月日，T出现在字符串中，表示time元素的开头，HH:mm:ss表示时分秒，TIMEZONE表示时区（+08:00表示东八区时间，领先UTC 8小时，即北京时间）。例如：2015-05-20T13:29:35+08:00表示，北京时间2015年5月20日 13点29分35秒。 |

| plan_list.stock_id | string(40) | false | 【投放批次ID】 投放的商品券批次ID，通过请求创建商品券接口获得，具体可查看 [创建商品券](https://pay.weixin.qq.com/doc/v3/partner/4016434630.md) ，当且仅当 usage\_mode 为 SINGLE 时必传。 |
| plan_list.stock_bundle_id | string | false | 【投放批次组ID】 投放的批次组ID，通过请求创建商品券接口获得，具体可查看 [创建商品券](https://pay.weixin.qq.com/doc/v3/partner/4016434630.md) ，当且仅当 usage\_mode 为 PROGRESSIVE\_BUNDLE 时必填。 |
| plan_list.recommend_word | string(27) | false | 【营销标签】 用于在优惠左上角展示的运营推荐语信息。自定义文案，不超过9个中文字符或18个英文字符。若创建时未设置则不返回。 |
| plan_list.brand_id | string(32) | true | 【品牌ID】 创建投放计划的品牌ID，可登录 [合作伙伴平台](https://pay.weixin.qq.com/index.php/partner/public/home) ，在“品牌经营中心-合作品牌管理”页面查看已授权的brand\_id信息 |
| plan_list.total_count | integer | true | 【投放总库存数量】 约定了投放计划的总投放库存。 |
| plan_list.user_limit | integer | true | 【单用户限领】 约定了投放计划单用户维度的限领数量； 非必填，如创建时未填写，则修改时不支持填写。 |
| plan_list.daily_limit | integer | true | 【单日限领】 用于约定投放计划单日可领取的最大数量，如创建时未填写，则修改时不支持填写。 |
| plan_list.reuse_coupon_config | boolean | true | 【是否复用商品券和批次信息】 是：表示从商品券和批次获取信息自动填充plan\_name、total\_count、user\_limit、daily\_limit、delivery\_start\_time、delivery\_end\_time。当投放计划在投放中状态时，若商品券批次的库存发生变化，投放计划会自动更新最新库存。<br>注：<br>plan\_name默认取投放商品券的name字段；<br>total\_count默认取投放批次的max\_count字段；<br>user\_limit默认取投放批次的max\_count\_per\_user字段；<br>daily\_limit默认取投放批次的max\_count\_per\_day字段；<br>delivery\_start\_time默认取商品券的available\_begin\_time字段；<br>delivery\_end\_time默认取批次的 available\_end\_time 字段并-1，若批次配置有 wait\_days\_after\_receive和available\_days信息，则delivery\_end\_time默认取值为 available\_end\_time - wait\_days\_after\_receive - available\_days -1。<br>否：表示自定义传入plan\_name、total\_count、user\_limit、daily\_limit、delivery\_start\_time、delivery\_end\_time，其中plan\_name、total\_count、delivery\_start\_time、delivery\_end\_time、user\_limit、daily\_limit必填 |
