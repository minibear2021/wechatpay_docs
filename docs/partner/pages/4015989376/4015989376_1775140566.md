# 1、产品概述

微信支付商品券是微信支付为品牌提供的优惠券全链路解决方案，支持商户通过API接口实现商品券的创建、发放、领取、通知、核销，券信息查询等配套功能完成商品券的管理操作，并支持将商品券用于平台流量场景与商家自有流量场景投放。

# 2、产品介绍

## 2.1、商品券模式

商品券分为单券模式和多次优惠模式。

- 单券模式：用户只能使用一次，使用后券失效。

- 多次优惠模式：用户可以按顺序多次使用，每次核销成功后会发放下一次优惠机会，直到用完为止。


本文档介绍“单券”模式，“多次优惠”模式可查看 [商品券(多次优惠)-产品介绍](https://pay.weixin.qq.com/doc/v3/partner/4016438787.md)

## 2.2、商品券（单券）结构

商品券由商品信息和批次组成：

- 商品信息：定义了商品信息与优惠模式，基础信息如商品券名称、图片等；优惠模式包括使用模式、适用范围、券类型等配置，一个商品券可以关联多个批次。

- 批次信息：是基于商品信息创建的优惠券批次，包含了实际的优惠规则信息、发放规则、使用规则等内容，一个批次仅能关联一个商品券。


![](https://gtimg.wechatpay.cn/resource/xres/mmpaydoc/static/img/284fa4949e43c308fecf9d2ef397b870.png)

产品特点

- 用户消费资产沉淀：所有消费行为（如购买、核销、评价）归集于同一商品券ID之下，帮助用户消费信息资产持续积累，为精准营销与用户消费转化提供数据资产基础。

- 品牌配置效率提升：针对共享相同商品信息的批次，可基于已有商品券直接创建新批次，无需重复配置商品信息，减少重复操作。

## 2.4、商品券券码类型

商品券券Code可由微信支付后台生成，同时也支持品牌自定义，两者的区别如下：

- 微信支付侧随机生成券code


商品券Code信息，发券时由微信支付系统自动随机生成券码，品牌方无需干预，微信支付随机生成的券Code长度限制为40个字符以内。

- 品牌侧自定义券Code


品牌方自定义券Code，更多适用于已有Code码体系做营销活动的品牌方。自定义券Code包含预存Code模式、发放时指定Code模式两种模式。

1、预存Code模式

品牌方预先在微信支付系统内调用[预上传券Code API](https://pay.weixin.qq.com/doc/brand/4015736400.md)上传自定义Code，微信支付系统发券时从已导入的code种随机选取（不能指定），派发给用户。当预存Code不足时可能影响发券，品牌应及时补充。

2、发放时指定Code模式

品牌方自行指定，品牌方通过[发券API](https://pay.weixin.qq.com/doc/v3/partner/4015781605.md)自行发券时指定，微信支付系统不再进行自动分配。特别注意，这种模式的商品批次只可由品牌方自行发券，无法在摇一摇有优惠等微信支付营销场景投放。

## 2.5、小程序发券组件功能介绍

微信支付提供的官方小程序发券插件。品牌商家的小程序接入此插件后，可在小程序内发放指指定批次的微信支付商品券。用户领券后，可收到消息触达，提升用户用券体验促进核销率

# 3、营销场景

商品券能够在微信内摇优惠活动、商家名片等场景分发，基于不同的券生命周期状态下发消息提醒。

## 3.1、摇一摇有优惠

[摇一摇有优惠](https://pay.weixin.qq.com/doc/v3/partner/4015782374.md)：微信支付推出的一种互动营销功能，基于用户在微信支付的消费偏好，向用户提供最短路径优惠「发放」（支付完成页下发优惠券）和「使用」优惠的能力，从而帮助商户实现交易闭环与增长。

![](https://gtimg.wechatpay.cn/resource/xres/mmpaydoc/static/img/0252c67bf25103ebc82c07b5d1e8d9ad.png)

## 3.2、商家名片

商家名片连接商家经营及服务。用户支付完成后，每一笔交易的支付凭证点击都可以跳转商家名片，同时，若商家有优惠，会在支付凭证下方cell外露，在用户产生每笔交易时提供用户领券用券。

在商家名片上展示商家优惠的几种方式：

- 待领取的优惠：通过[创建投放计划](https://pay.weixin.qq.com/doc/v3/partner/4016111064.md)，有机会在商家名片中曝光优惠，用户可点击领取。

- 待使用的优惠：


1、商家通过「[向用户发放商品券](https://pay.weixin.qq.com/doc/v3/partner/4015781605.md)」接口发放的券，会展示在「待使用」的优惠中，同时也会在支付凭证上外显

2、用户通过「摇一摇有优惠」领取的券

3、用户通过「支付有礼」领取的券

同时，商家名片是微信支付为商家提供的会员经营工具，若配置[商家名片会员](https://pay.weixin.qq.com/doc/v3/partner/4015274636.md)相关能力，可提供联动微信支付下单的会员资产变动感知，拉动入会，跳转商家小程序会员相关页面，发放会员券的能力，结合微信生态能力向品牌商家提供更丰富的会员解决方案，帮助商家提升会员运营效率。

![](https://gtimg.wechatpay.cn/resource/xres/mmpaydoc/static/img/99c0c6a19a37c919bbc8356f2df7eb87.png)

## 3.3、消息提醒

基于商品券的生命周期状态（如核销、到期），系统将自动通过微信支付公众号向用户推送提醒，直接触达用户，提升运营效率。
