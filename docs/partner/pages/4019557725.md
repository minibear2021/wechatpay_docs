>更新时间：2026.06.29

对商户提供同步车辆离场通知接口




## 接口说明

支持商户：【普通服务商】　【平台商户】

请求方式：【POST】 `/v3/parking/reminders/exit`

请求域名：【主域名】https://api.mch.weixin.qq.com 使用该域名将访问就近的接入点

　　　　　【备域名】https://api2.mch.weixin.qq.com 使用该域名将访问异地的接入点，指引[点击查看](https://pay.weixin.qq.com/doc/v3/merchant/4012075113.md)

## 请求参数

### Header 参数
| 参数名 | 类型 | 必填 | 描述 |
| --- |  --- | --- | --- |
| Authorization | string | true | 请参考 [签名认证](https://pay.weixin.qq.com/doc/v3/merchant/4012365342.md#%E8%AF%B7%E6%B1%82%E6%8E%A5%E5%8F%A3%E5%90%8E%E7%AB%AF%E7%AD%BE%E5%90%8D) 生成认证信息 |
| Accept | string | true | 请设置为 `application/json` |
| Content-Type | string | true | 请设置为 `application/json` |
### Body 参数
| 参数名 | 类型 | 必填 | 描述 |
| --- |  --- | --- | --- |
| out_serial_number | string(64) | true | 【商户停车单号】 商户系统自行生成，须在同一商户号下全局唯一。建议不超过32字符，超过64字符将被拦截。与入场时同步的 out\_serial\_number 保持一致。 |
| exit_timestamp | integer | true | 【车辆出场时间】 秒级时间戳 |
| plate_number | string(16) | true | 【车牌号】 国内车牌，省份简称+6~7位字母或数字（新能源绿牌8位）。 |
| plate_color | string | true | 【车牌颜色】 按照本文档提供的车牌颜色枚举值传入，各枚举值含义见字段枚举说明。<br>可选取值<br><ul><li>`PLATE_COLOR_UNKNOWN`:  未知</li><li>`PLATE_COLOR_BLUE`:  蓝色</li><li>`PLATE_COLOR_YELLOW`:  黄色</li><li>`PLATE_COLOR_BLACK`:  黑色</li><li>`PLATE_COLOR_WHITE`:  白色</li><li>`PLATE_COLOR_GREEN`:  绿色</li><li>`PLATE_COLOR_YELLOW_GREEN`:  黄绿</li><li>`PLATE_COLOR_AGRICULTURAL_YELLOW`:  农黄</li><li>`PLATE_COLOR_OTHER`:  其他</li></ul> |
| parking_id | string(64) | true | 【微信停车场ID】 停车场进件审核通过且运营入驻完成后，由微信支付分配。可通过「查询停车场进件申请单」或「查询停车场信息」接口获取；入场/离场/支付等接口须与入场时保持一致。须与入场时同步的微信停车场ID保持一致，不一致将报错。 |
| pay_state | string | false | 【支付状态】 仅在通过“同步车辆离场通知”接口同步支付结果时，必须填写该字段；如果通过“同步支付结果通知”接口同步支付结果，则无需填写；也可以“同步支付结果通知”接口和“同步车辆离场通知”接口都同步支付结果，但都要满足我们这里的字段规范要求，注意字段的必填性<br>可选取值<br><ul><li>`PAY_STATUS_UNKNOWN`:  未知</li><li>`PAY_STATUS_UNPAID`:  未支付</li><li>`PAY_STATUS_PAID`:  已支付</li></ul> |
| pay_type | string | false | 【支付类型】 当支付状态为已支付时必填，且仅支持微信支付（PAY\_TYPE\_WXPAY）<br>可选取值<br><ul><li>`PAY_TYPE_UNKNOWN`:  未知</li><li>`PAY_TYPE_WXPAY`:  微信支付</li><li>`PAY_TYPE_OTHERS`:  其他支付</li></ul> |
| out_trade_no | string(64) | false | 【商户支付订单号】 商户侧支付订单号。已支付时，与 wx\_trade\_no 至少二选一必传，建议全填。长度1~64字符，同一商户号下建议唯一。 |
| total_amount | integer | false | 【总费用 】 单位为分，已支付状态时，必填 |
| paid_amount | integer | false | 【已支付费用 】 单位为分，已支付状态时，必填；当支付类型为微信支付时不允许为零 |
| openid | string(128) | false | 【付款用户OpenID】 支付渠道为微信支付时必填。获取方式详见 [参数说明](https://pay.weixin.qq.com/doc/v3/partner/4012081935.md)。已支付状态且为微信支付时必填。 |
| pay_channel | string | false | 【支付渠道】 已支付状态下建议填写，用于标识支付来源。<br>可选取值<br><ul><li>`PAY_CHANNEL_UNKNOWN`:  未知</li><li>`PAY_CHANNEL_WECHAT_PAY_NOTIF`:  微信支付通知消息</li><li>`PAY_CHANNEL_SCAN_QRCODE`:  扫码</li><li>`PAY_CHANNEL_OFFICIAL_ACCOUNT`:  公众号</li><li>`PAY_CHANNEL_MCH_MINI_PROG`:  商户小程序</li><li>`PAY_CHANNEL_OTHERS`:  其他</li></ul> |
| pay_time | integer | false | 【支付时间】 支付完成时刻，秒级时间戳（Unix timestamp）。例如：1719128975 表示北京时间2024-06-23 13:29:35。 |
| token | string(256) | false | 【消息支付防伪凭证】 原样回传微信支付随缴费入口下发的凭证；离场携带支付信息时可选回传，用于服务端验真消息支付，缺省即未回传。长度1~256字符。 |
| wx_trade_no | string(64) | false | 【微信支付订单号】 微信支付订单号。已支付时，与 out\_trade\_no 至少二选一必传，建议全填。长度1~64字符。 |
