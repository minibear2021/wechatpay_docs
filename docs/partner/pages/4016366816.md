接口适用场景：服务商申请添加交易连接名片规则后，可调用此接口查询申请状态。




## 接口说明

支持商户：【普通服务商】

请求方式：【GET】 `/v3/brand/card/card-links/business-code/{business_code}`

请求域名：【主域名】https://api.mch.weixin.qq.com 使用该域名将访问就近的接入点

　　　　　【备域名】https://api2.mch.weixin.qq.com 使用该域名将访问异地的接入点，指引[点击查看](https://pay.weixin.qq.com/doc/v3/merchant/4012075113.md)

接口限频：100/秒（商户号维度）

## 应答参数

| 参数名 | 类型 | 必填 | 描述 |

| brand_id | string(20) | true | 【品牌ID】 1、商家进驻微信支付品牌商家后获得的品牌ID，作为品牌唯一标识符；<br>2、数字，长度不超过 20 个字符;<br>3、暂不支持银行品牌。 |
| payment_scene | string | true | 【交易场景类型】 交易场景类型是指微信支付提供的帮助商家在不同场景中快速收款的工具（如小程序支付、付款码支付等）。<br>不同类型的交易场景下，建立交易连接名片规则依据的账号类型不同：<br>1、当枚举值选择 `MINI_PROGRAM`、 `APP` 时， 返回 `appid` 字段；<br>2、当枚举值选择 `PAYMENT_SCORE` 时，返回 `service_id` 字段；<br>3、当枚举值选择 `PAYMENT_CODE` 时， 返回 `card_link_mchid` 字段。<br>可选取值<br><ul><li>`MINI_PROGRAM`:  小程序支付：用户在品牌的小程序中选择商品下单，跳转微信支付确认验密页面，完成支付，详细可查看 [产品介绍](https://pay.weixin.qq.com/doc/v3/partner/4012085810.md)</li><li>`APP`:  APP 支付：用户在品牌的App中选择商品下单，跳转到微信支付确认验密页面，完成支付，详细可查看 [产品介绍](https://pay.weixin.qq.com/doc/v3/partner/4013080227.md)</li><li>`PAYMENT_SCORE`:  支付分支付：微信支付分提供先使用服务，再付款的能力，如免押金借用充电宝、电商先用后付等，详细可查看 [产品介绍](https://pay.weixin.qq.com/doc/v3/partner/4012586132.md)</li><li>`PAYMENT_CODE`:  付款码支付：用户线下展示付款码，商户收银员用扫码设备扫描用户的付款码，用户确认支付，详细可查看 [场景介绍](https://pay.weixin.qq.com/doc/v2/partner/4011941052.md)</li></ul> |
| appid | string(20) | false | 【公众账号ID】 公众账号ID也称 APPID，是（微信开放平台、微信公众平台）为开发者提供的一个唯一标识，用于识别开发者的应用程序（APP、小程序、公众号）。<br>1、若交易场景类型为小程序支付、APP支付时，此字段返回对应的 AppID；<br>2、其他交易场景类型时，此字段返回空值。 |
| card_link_mchid | string(20) | false | 【商户号】 商户号是商户在 [商户平台](https://pay.weixin.qq.com/index.php/core/home/login?return_url=%2F) 申请的唯一身份标识，微信支付通过商户号确认商户身份。<br>1、当交易场景为付款码支付，或小程序支付且对应的 appid 为收银类平台小程序时，此字段返回对应的商户号；<br>2、其他交易场景类型时，此字段返回空值。 |
| service_id | string(50) | false | 【服务ID】 支付分服务ID是商户支付分服务的唯一标识，由32位数字组成。支付分产品权限申请审核通过后，微信支付运营会向商户提供支付分服务ID。<br>1、当交易场景类型为支付分支付时，此字段返回对应的支付分服务ID；<br>2、其他交易场景类型时，此字段返回空值。 |
| configuration_state | string | true | 【交易连接名片配置状态】 该字段返回交易连接名片的配置状态。<br>可选取值<br><ul><li>`STATE_UNKNOWN`:  「状态未知，非终态」连接规则建立中，请稍后</li><li>`WAITING_AUDIT`:  「审核中，非终态」创建交易链路请求受理后，平台将进行审核，即流转到该状态</li><li>`AUDIT_REJECT`:  「审核驳回，终态」审核阶段被驳回，会附带具体的驳回原因，请根据驳回原因修改调整，如仍需添加请使用原业务申请编号重新调用 [配置交易连接名片](https://pay.weixin.qq.com/doc/v3/partner/4016333302.md) 接口</li><li>`WAITING_CONFIRMATION`:  「待商户号超管确认，非终态」添加付款码支付/收银类小程序支付场景时，若提供的商户号主体与品牌不一致，则需商户号超管确认才能完成规则配置，请提醒对方超管查看「微信支付商家助手」继续确认</li><li>`MERCHANT_ADMIN_REJECT`:  「商户号超管已拒绝，终态」付款码支付/收银类小程序支付场景，绑定异主体商户号，商户号超管已拒绝或超时，如仍需绑定请使用原业务申请编号重新调用 [配置交易连接名片](https://pay.weixin.qq.com/doc/v3/partner/4016333302.md) 接口</li><li>`IN_EFFECT`:  「生效中，终态」交易链路建立成功</li><li>`CANCELED`:  「已撤销，终态」交易链路请求被撤销，如仍需添加请更换业务申请编号重新调用 [配置交易连接名片](https://pay.weixin.qq.com/doc/v3/partner/4016333302.md) 接口</li></ul> |
