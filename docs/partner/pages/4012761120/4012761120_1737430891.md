服务商/电商平台（不包括支付机构、银行），可使用本接口，查询其进件且已签约特约商户/二级商户的结算账户的修改申请单信息和审核结果。

注意：

1、查询结算账户修改申请状态接口调用频率限制：100/秒。




## 接口说明

支持商户：【普通服务商】　【平台商户】

请求方式：【GET】 `/v3/apply4sub/sub_merchants/{sub_mchid}/application/{application_no}`

请求域名：【主域名】https://api.mch.weixin.qq.com 使用该域名将访问就近的接入点

　　　　　【备域名】https://api2.mch.weixin.qq.com 使用该域名将访问异地的接入点，指引[点击查看](https://pay.weixin.qq.com/doc/v3/partner/4012082567.md)

## 请求参数

### Header 参数
| 参数名 | 类型 | 必填 | 描述 |
| --- |  --- | --- | --- |
| Authorization | string | true | 请参考 [签名认证](https://pay.weixin.qq.com/doc/v3/partner/4012365870.md#%E8%AF%B7%E6%B1%82%E6%8E%A5%E5%8F%A3%E5%90%8E%E7%AB%AF%E7%AD%BE%E5%90%8D) 生成认证信息 |
| Accept | string | true | 请设置为 `application/json` |
### Path 参数
| 参数名 | 类型 | 必填 | 描述 |
| --- |  --- | --- | --- |
| sub_mchid | string(32) | true | 【特约商户/二级商户号】 请填写本服务商负责进件的特约商户/二级商户号。 |
| application_no | string(64) | true | 【修改结算账户申请单号】 提交二级商户修改结算账户申请后，由微信支付返回的单号，作为查询申请状态的唯一标识。 |
### Query 参数
| 参数名 | 类型 | 必填 | 描述 |
| --- |  --- | --- | --- |
| account_number_rule | string | false | 【银行账号展示规则】 请输入预期的银行账号展示规则，默认为ACCOUNT\_NUMBER\_RULE\_MASK\_V1<br>可选取值<br><ul><li>`ACCOUNT_NUMBER_RULE_MASK_V1`: 银行账号展示规则掩码v1，保留前2位和最后2位，中间用\*代替</li><li>`ACCOUNT_NUMBER_RULE_MASK_V2`: 银行账号展示规则掩码v2，保留前6位和最后4位，中间用\*代替</li></ul> |
## 应答参数

200 OK

| 参数名 | 类型 | 必填 | 描述 |
| --- |  --- | --- | --- |
| account_name | string(1024) | true | 【开户名称】 特约商户的结算账户-开户名称，掩码显示。 |
| account_type | string | true | 【账户类型】 特约商户的结算账户类型。<br>可选取值<br><ul><li>`ACCOUNT_TYPE_BUSINESS`:  对公银行账户</li><li>`ACCOUNT_TYPE_PRIVATE`:  经营者个人银行卡</li></ul> |
| account_bank | string(128) | true | 【开户银行】 特约商户的结算账户-开户银行全称。 |
| bank_name | string(128) | false | 【开户银行全称（含支行）】 特约商户的结算账户-开户银行全称（含支行）。 |
| bank_branch_id | string(128) | false | 【开户银行联行号】 特约商户的结算账户-联行号。 |
| account_number | string(128) | true | 【银行账号】 特约商户的结算账户-银行账号，根据account\_number\_rule指定规则展示。 |
| verify_result | string | true | 【审核状态】 审核状态<br>可选取值<br><ul><li>`AUDIT_SUCCESS`:  审核成功</li><li>`AUDITING`:  审核中</li><li>`AUDIT_FAIL`:  审核驳回</li></ul> |
| verify_fail_reason | string | false | 【审核驳回原因】 审核驳回原因。审核成功时为空，审核驳回时为具体原因。 |
| verify_finish_time | string | false | 【审核结果更新时间】 遵循 [rfc3339](https://datatracker.ietf.org/doc/html/rfc3339) 标准格式，格式为yyyy-MM-DDTHH:mm:ss+TIMEZONE，yyyy-MM-DD表示年月日，T出现在字符串中，表示time元素的开头，HH:mm:ss表示时分秒，TIMEZONE表示时区（+08:00表示东八区时间，领先UTC 8小时，即北京时间）。例如：2015-05-20T13:29:35+08:00表示，北京时间2015年5月20日13点29分35秒。 |
