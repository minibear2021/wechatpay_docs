当服务商提交申请单后，需要定期调用此接口查询申请单的审核状态。

接口频率：20QPS(商户号维度)




## 接口说明

支持商户：【渠道商】　【从业机构（银行）】　【从业机构（支付机构）】

请求方式：【GET】 `/v3/apply4subject/applyment`

请求域名：【主域名】https://api.mch.weixin.qq.com 使用该域名将访问就近的接入点

　　　　　【备域名】https://api2.mch.weixin.qq.com 使用该域名将访问异地的接入点，指引[点击查看](https://pay.weixin.qq.com/doc/v3/partner/4012082567.md)

## 应答参数

200 OK

| 参数名 | 类型 | 必填 | 描述 |
| --- |  --- | --- | --- |
| applyment_state | string(64) | true | **【申请单状态】**<br>可选取值：：<br>- `APPLYMENT_STATE_EDITTING`: 如果查到申请单是编辑中的状态，可能是提交申请发生了错误导致的，可用同一个业务申请编号重新提交<br>- `APPLYMENT_STATE_WAITTING_FOR_AUDIT`: 如果查到申请单是审核中的状态，请耐心等待1~2个工作日，微信支付将会完成审核<br>- `APPLYMENT_STATE_WAITTING_FOR_CONFIRM_CONTACT`: 如果查到申请单是待确认联系信息的状态，请扫描微信支付返回的小程序码确认联系信息(此过程可修改联系人手机号)<br>- `APPLYMENT_STATE_WAITTING_FOR_CONFIRM_LEGALPERSON`: 如果查到申请单是待账户验证的状态，请扫描微信支付返回的小程序码在小程序端完成账户验证<br>- `APPLYMENT_STATE_PASSED`: 如果查到申请单是审核通过状态，请扫描微信支付返回的小程序码在小程序端完成授权流程<br>- `APPLYMENT_STATE_REJECTED`: 如果查到申请单是审核驳回状态，请按照驳回原因修改申请资料，并更换业务申请编码，重新提交申请<br>- `APPLYMENT_STATE_FREEZED`: 如果查到申请单是已冻结状态，可能是该主体已完成过入驻，请查看驳回原因，并通知驳回原因中指定的联系人扫描微信支付返回的小程序码在小程序端完成授权流程<br>- `APPLYMENT_STATE_CANCELED`: 如果查到申请单是已作废状态，表示申请单已被撤销，无需再对其进行操作 |
| qrcode_data | string(30000) | false | 【小程序码图片】 1、当申请单状态为APPLYMENT\_STATE\_WAITTING\_FOR\_CONFIRM\_CONTACT、APPLYMENT\_STATE\_WAITTING\_FOR\_CONFIRM\_LEGALPERSON、APPLYMENT\_STATE\_PASSED、APPLYMENT\_STATE\_FREEZED时，会返回小程序码图片<br>2、使用base64解码该字段，可得到图片二进制数据<br>3、可用img标签直接加载该图片，示例如下：<img src="data:image/png;base64,iVBORw0KGgoAAAANSU=" style="display: block;"> |
| reject_param | string(256) | false | **【驳回参数】** 当申请单状态为APPLYMENT\_STATE\_REJECTED、APPLYMENT\_STATE\_FREEZED时，会返回该字段，标识被驳回的字段名 |
| reject_reason | string(1024) | false | **【驳回原因】** 当申请单状态为APPLYMENT\_STATE\_REJECTED、APPLYMENT\_STATE\_FREEZED时，会返回该字段，表示驳回原因 |
