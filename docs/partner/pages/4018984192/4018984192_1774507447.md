调用商品券相关接口时，根据接入模式的不同（分为：品牌模式、服务商模式），分别需要品牌与appid进行关联，以及服务商与appid进行关联，本文主要用于说明两者的关联方法。

## 一、三者关联关系基础

| 关联主体 | 关联简称 | 关系类型 | 说明 |
| :-- | :-- | :-- | :-- |
| mchid ↔ brand\_id | BM | 多对多 | 一个服务商（mchid）可服务多个品牌（brand\_id），一个品牌也可授权给多个服务商。<br>仅服务商模式下需建立授权关系，授权流程请参考：[服务商申请品牌授权流程](https://pay.weixin.qq.com/doc/v3/partner/4015788437.md) |
| mchid ↔ appid | MA | 多对多 | 一个服务商下可关联多个appid，一个appid也可关联多个服务商 |
| brand\_id ↔ appid | BA | 多对多 | 一个品牌下可关联多个appid，一个appid也可关联多个品牌 |

![](https://gtimg.wechatpay.cn/resource/xres/mmpaydoc/static/img/913269cfdbef39d42589f7329479adbb.png)

## 二、appid认证

未进行认证的appid将无法关联品牌（brand\_id）、服务商（mchid），需先参考以下流程完成验证：

- 服务号、政府或媒体类型的公众号：参考[服务号/公众号认证指引](https://kf.qq.com/product/weixinmp.html#hid=99)完成公众平台账号认证；

- 小程序：参考[小程序认证指引](https://kf.qq.com/faq/170109by2mQB170109zUFfqe.html)完成公众平台账号认证；

- 移动应用：参考[开放平台账号认证指引](https://kf.qq.com/faq/170824URbmau170824r2uY7j.html)完成开放平台账号认证。


## 三、品牌（brand\_id）关联appid

品牌关联appid目前有两种路径：

- 路径一（推荐）：小程序、服务号、公众号、APP类型的都可以按以下步骤进行关联：

  - 步骤1：品牌方在【品牌经营平台-账号管理-下单appid管理】页面，点击“关联appid”。

  - 步骤2：品牌经营平台完成关联后，将在次日生效，请耐心等待。

![](https://gtimg.wechatpay.cn/resource/xres/mmpaydoc/static/img/3ce5541b52c36adeb1c6d49ebe54673e.png)

- 路径二：小程序、APP类型的appid，如果是用于支付交易的appid，可通过“交易连接名片”功能进行关联，两种关联方式：

  - 方式一：服务商通过商家名片下的“[交易连接名片](https://pay.weixin.qq.com/doc/v3/partner/4016985845.md)”功能，为品牌方关联appid。（目前只支持服务商模式）

  - 方式二：品牌方在【品牌经营平台-商家名片-交易链接名片】页面，点击“添加”关联appid。

![](https://gtimg.wechatpay.cn/resource/xres/mmpaydoc/static/img/3776e137fffea5368b2935f41966911a.png)

注意：

路径二涉及的“交易连接名片”功能并非专门用于建立BA关系，而是一个单独的产品功能，该功能会存在限制：一个appid只能和一个品牌的“交易连接名片”关联，不能同时去关联其他品牌的“交易连接名片”。

但该限制不影响appid通过路径一去关联其他品牌，因此推荐通过路径一让品牌关联appid。

## 四、服务商（mchid）关联appid

服务商可登录[服务商平台](https://pay.weixin.qq.com/index.php/partner/public/home)，通过路径【产品中心 -\> APPID账号管理 -> 我关联的APPID账号】来查看所有绑定的APPID账号。

在该页面可以新增关联appid，具体流程请参考[服务商商户号与AppID账号关联管理](https://kf.qq.com/faq/200520fueQrI200520aQf6Rr.html)。

![](https://gtimg.wechatpay.cn/resource/xres/mmpaydoc/static/img/efd1aa570c8665e47bf1abce12456a9a.png)

## 五、关联关系的应用

| 模式 | 应用说明 |
| :-- | :-- |
| 品牌模式 | 品牌模式的商品券接口只需建立BA关系，接口中填写的appid相关参数均为品牌关联的appid。 |
| 服务商模式 | 服务商模式的商品券接口必须先建立BM关系才能调用，接口中大部分appid相关参数同时支持BA、MA关系，商户侧可按需选择。 |
| 若使用未关联的appid调用接口，会返回报错：AppID非法 |
