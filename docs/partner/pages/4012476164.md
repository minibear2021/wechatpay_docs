电商平台发起申请后, 电商平台调用此接口查询审批和出款进度




## 接口说明

支持商户：【平台商户】

请求方式：【GET】 `/v3/mch_operate/risk/withdrawl-apply/out-request-no/{out_request_no}`

请求域名：【主域名】https://api.mch.weixin.qq.com 使用该域名将访问就近的接入点

　　　　　【备域名】https://api2.mch.weixin.qq.com 使用该域名将访问异地的接入点，指引[点击查看](https://pay.weixin.qq.com/doc/v3/partner/4012082567.md)

### Path 参数
| 参数名 | 类型 | 必填 | 描述 |
| --- |  --- | --- | --- |
| out_request_no | string(32) | true | **【商户提现申请单号】** 由商户自定义生成的、在提交接口传入的单号 |
## 应答参数

200 OK

| 参数名 | 类型 | 必填 | 描述 |
| --- |  --- | --- | --- |
| withdrawl_apply | object | true | **【提现申请单信息】** 提现申请单信息 |
| withdrawl_apply.applyment_id | string(32) | true | **【微信支付提现申请单号】** 电商平台提交二级商户可用余额提现申请后，由微信支付返回的申请单号，作为查询申请状态的唯一标识。 |
| withdrawl_apply.out_request_no | string(32) | true | **【商户提现申请单号】** 商户预约提现单号，由商户自定义生成，需要全局唯一 |
| withdrawl_apply.state | string | true | **【申请单状态】** 状态<br>可选取值：<br><ul><li>`SUBMITTED`: 等待审核</li></ul><br><ul><li>`PASSED`: 付款执行中</li></ul><br><ul><li>`PAY_SUCCEED`: 付款成功</li></ul><br><ul><li>`REJECTED`: 审批驳回</li></ul><br><ul><li>`PAY_FAILED`: 付款失败</li></ul><br><ul><li>`BANK_REFUNDED`: 银行退票</li></ul> |
| withdrawl_apply.fail_reason | string(1024) | false | **【失败原因】** 包含付款关闭原因、付款失败原因、付款退票原因 |
| withdrawl_apply.modify_time | string(32) | false | **【最后更新时间】** 遵循 [rfc3339](https://datatracker.ietf.org/doc/html/rfc3339) 标准格式，格式为yyyy-MM-DDTHH:mm:ss+TIMEZONE，yyyy-MM-DD表示年月日，T出现在字符串中，表示time元素的开头，HH:mm:ss表示时分秒，TIMEZONE表示时区（+08:00表示东八区时间，领先UTC 8小时，即北京时间）。例如：2015-05-20T13:29:35+08:00表示，北京时间2015年5月20日 13点29分35秒。 |
