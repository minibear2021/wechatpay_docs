## 1. 概述

接入微信支付分的商户需在创建、完结、修改订单金额时，通过API接口中的post\_payments参数回传详细的后付费项目明细。该明细将在支付分订单详情页面展示，减少用户对订单金额的疑问。

post\_payments为数组类型参数，该参数可包含多条“后付费项目明细”，至少传1条明细，最多可传100条明细，每条“后付费项目明细”包含name、amount、description、count四个参数。

## 2. 后付费项目参数传值要求

### 2.1 创建支付分订单接口

快递行业的商户在[创建支付分订单](https://pay.weixin.qq.com/doc/v3/partner/4013138534.md)时，需通过post\_payments【后付费项目】参数回传订单的预估费用，以便于在支付分订单明细中向用户展示。

post\_payments参数传值要求：

|     |     |     |     |     |     |     |
| --- | --- | --- | --- | --- | --- | --- |
| 字段简称 | 字段名 | 类型\[长度限制\] | 是否必填 | 示例值 | 描述 | 传值示例 |
| 后付费项目名称 | name | string \[1, 20\] | 必填 | 预估金额 | 付费项目名称，请填写“预估金额”四个字 | "post\_payments" :  \[{<br>    "name" :   "预估金额",<br>    "amount" : 1200<br>}\] |
| 后付费项目金额 | amount | int | 必填 | 1200 | 付费项目金额，单位为分 |
| 后付费项目说明 | description | string \[1, 30\] | 不填 | - | 请勿传该字段 |
| 后付费项目的数量 | count | int | 不填 | - | 请勿传该字段 |

### 2.2 完结支付分订单接口

当服务结束后，商户调用[完结支付分订单](https://pay.weixin.qq.com/doc/v3/partner/4013138598.md)接口，需通过post\_payments【后付费项目】参数回传订单实际的付费项目，在快递行业中，只允许回传“首重费、续重费、包装费、保价费”这4类付费项目。

post\_payments必须包含一笔“首重费”的后付费项目；若实际收费中，还有“续重费”“包装费”“保价费”，请按照实际情况添加后付费项目明细。

例如：如果只有一笔后付费项目，只能传“首重费”，如果有多笔后付费项目，则必须包含一笔“首重费”，其他费用按照实际填写，详见传值示例。

post\_payments参数传值要求：

|     |     |     |     |     |     |     |     |
| --- | --- | --- | --- | --- | --- | --- | --- |
| 字段简称 | 字段名 | 类型\[长度限制\] | 是否必填 | 示例值 | 描述 | 传值示例(只有首重费) | 传值示例(含有首重费、续重费) |
| 后付费项目名称 | name | string\[1, 20\] | 必填 | 首重费 | 付费项目名称，只可填写如下取值<br>可选取值：<br>    首重费、续重费、包装费、保价费 | "post\_payments" :  \[{<br>    "name" :   "首重费",<br>    "amount" : 1200<br>}\] | "post\_payments" :  \[{<br>    "name" :   "首重费",<br>    "amount" : 1200<br>},{<br>    "name" :   "续重费",<br>    "amount" : 1200<br>}<br>\] |
| 后付费项目金额 | amount | int | 必填 | 1200 | 付费项目金额，单位为分 |
| 后付费项目说明 | description | string \[1, 30\] | 不填 | - | 请勿传该字段 |
| 后付费项目的数量 | count | int | 不填 | - | 请勿传该字段 |

后付费项目明细在支付分订单页面的实际展示效果（以下两种呈现形式为版本差异，均符合预期）：

### 2.3 修改订单金额接口

完结支付分订单后，且订单还处于“待支付（USER\_PAYING）”状态时，可调用[修改支付分订单金额](https://pay.weixin.qq.com/doc/v3/partner/4013138819.md)接口“下调”订单的金额，传值要求与完结支付分订单接口传值要求一致。
