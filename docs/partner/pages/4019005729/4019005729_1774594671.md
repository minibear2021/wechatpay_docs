商品券的发券渠道目前主要有3种常见方式：

- 商户主动调用商品券的API接口（不推送任何类型消息）

- 商户主动调用商品券的发券组件发放

- 商户配置投放计划，微信通过摇一摇有优惠功能发券


用户领券后，微信侧会根据券的状态，在客户端的“微信支付”账号中，向用户推送商品券相关消息，本文主要展示相关示例图。（消息的样式可能调整，本文仅作示例参考）

如果是通过摇一摇有优惠领取的券，消息会带有摇优惠的logo。

## 一、消息类型说明

|     |     |
| --- | --- |
| 消息类型 | 说明 |
| 领取消息 | 用户成功领取商品券后（券状态变为有效），向用户推送领券成功的消息。<br>注意：通过摇一摇有优惠领取的券，无该类型消息提醒 |
| 待使用提醒 | 用户成功领取商品券后，提醒用户待使用的券数量。<br>注意：仅“多次优惠”模式有该类型消息提醒 |
| 即将过期提醒消息 | 用户领取的券距离过期还有3天时，会向用户发送券即将过期的提醒。<br>注意：该类型提醒每天只会推送1次，且为各发放渠道所有券共用次数，若当天已有券触发该类型提醒，其他券不会再重复触发 |
| 核销消息 | 商户侧核销用户商品券成功后，向用户推送券已使用的消息。<br>注意：推送前提为商户侧上传的核销订单信息为微信支付真实订单，且在订单支付成功后的3分钟之内核销 |
