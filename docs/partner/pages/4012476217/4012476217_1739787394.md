电商子商户需注销商户号时，由其所属电商平台服务商调用本接口发起注销申请。




## 接口说明

支持商户：【平台商户】

请求方式：【POST】 `/v3/ecommerce/account/cancel-applications`

请求域名：【主域名】https://api.mch.weixin.qq.com 使用该域名将访问就近的接入点

　　　　　【备域名】https://api2.mch.weixin.qq.com 使用该域名将访问异地的接入点，指引[点击查看](https://pay.weixin.qq.com/doc/v3/partner/4012082567.md)

| out_apply_no | string(32) | true | 【商户注销申请单号】商户注销申请单号，由商户自定义生成，要求在服务商维度下是唯一的，必须仅包含大小写字母与数字 |
| application_info | array[CancelApplicationInfo] | true | 【注销申请材料】注销申请材料，详见下方文档：<br>[注销申请材料](https://docs.qq.com/doc/p/cc53548f88238e7d58337f733b4e2997c4fb3253) |
| application_info.application_type | string | true | 【注销申请材料类型】注销申请材料类型，详见下方文档：<br>[注销申请材料](https://docs.qq.com/doc/p/cc53548f88238e7d58337f733b4e2997c4fb3253)<br>可选取值：<br><ul><li>`SP_MERCHANT_APPLICATION`: 此枚举值已废弃，请使用新字段 \*SP\_CANCEL\_ACCOUNT\_APPLICATION\* 以及新版本材料</li></ul><br><ul><li>`SUB_MERCHANT_APPLICATION`: 此枚举值已废弃，请使用新字段 \*SUB\_CANCEL\_ACCOUNT\_APPLICATION\* 以及新版本材料</li></ul><br><ul><li>`MISSING_OFFICIAL_SEAL_LETTER`: 此材料已废弃，无需上传</li></ul><br><ul><li>`SP_CANCEL_ACCOUNT_APPLICATION`: 电商服务商注销电商子申请书，请下载模板打印纸质版、填写盖章后拍照。模板文档详见：<br>[微信支付商户号注销申请书-服务商（纸质版）](https://docs.qq.com/doc/p/2ad500a5fb5d1e1216b15f35fce0fd84642b0204)</li></ul><br><ul><li>`SUB_CANCEL_ACCOUNT_APPLICATION`: 电商服务商子商户注销申请书，详见文档：<br>[微信支付商户号注销申请书-电商平台子商户适用（纸质版）](https://docs.qq.com/doc/p/16a704190df2780bbc6e05d2c531ce2f38749870)</li></ul> |
| application_info.application_media_id | string(512) | true | 【注销申请材料照片ID】注销申请材料照片ID，请填写通过 [上传图片](https://pay.weixin.qq.com/doc/v3/partner/4012691710.md) 接口预先上传图片生成好的media\_id |
## 应答参数

200 OK

| 参数名 | 类型 | 必填 | 描述 |
| --- |  --- | --- | --- |
| out_apply_no | string(32) | false | 【商户注销申请单号】商户注销申请单号，原样返回请求参数里的内容 |
| sub_mchid | string(32) | false | 【二级商户号】二级商户号 |
| reject_reason | string(512) | false | 【驳回原因】受理失败原因 |
| cancel_state | string | false | 【注销状态】注销状态<br>可选取值：<br><ul><li>`REVIEWING`: 审核中</li></ul><br><ul><li>`REJECTED`: 审核驳回，驳回原因详见reject\_reason</li></ul><br><ul><li>`CANCEL_SUCCESS`: 注销成功</li></ul> |
| update_time | string(32) | false | 【最后更新时间】最后更新时间。遵循rfc3339标准格式，格式为yyyy-MM-DDTHH:mm:ss+TIMEZONE，yyyy-MM-DD表示年月日，T出现在字符串中，表示time元素的开头，HH:mm:ss表示时分秒，TIMEZONE表示时区（+08:00表示东八区时间，领先UTC 8小时，即北京时间）。例如：2015-05-20T13:29:35+08:00表示，北京时间2015年5月20日 13点29分35秒。 |
