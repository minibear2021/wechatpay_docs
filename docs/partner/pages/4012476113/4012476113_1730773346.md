商户发起资金出境请求，需要传微信支付单号，商户出境单号，出境金额等信息 接口请求成功仅代表受理成功，如需知晓业务执行情况请通过查询接口获取。




## 接口说明

支持商户：【平台商户】

请求方式：【POST】 `/v3/funds-to-oversea/orders`

请求域名：【主域名】https://api.mch.weixin.qq.com 使用该域名将访问就近的接入点

　　　　　【备域名】https://api2.mch.weixin.qq.com 使用该域名将访问异地的接入点，指引[点击查看](https://pay.weixin.qq.com/doc/v3/partner/4012082567.md)

## 应答参数

200 OK

| 参数名 | 类型 | 必填 | 描述 |
| --- |  --- | --- | --- |
| out_order_id | string(64) | true | **【商户出境单号】** 订单的主键，唯一定义此资源的标识，此参数只能由数字，大小写字母\_-组成。<br>由商户在发起资金出境请求时生成，要求在同一个商户号下唯一。 |
| sub_mchid | string(64) | true | **【二级商户号】** 申请资金出境的二级商户号 |
| order_id | string(128) | true | **【微信出境单号】** 微信出境单号 |
| result | string(128) | true | **【出境结果】** 出境的结果， 枚举值：<br>- ACCEPT：已受理<br>- SUCCESS：出境成功<br>- FAIL：出境失败 |
| fail_reason | string(128) | false | **【出境失败的原因】** 当result为FAIL时，会出现此字段，标明出境失败原因，如果是SYSTEM\_ERROR可以重新发起重试<br>失败原因：<br>- MCHID\_FROZEN：商户已冻结，转账失败<br>- DEAL\_TIMEOUT：单据已过期<br>- TRADE\_SUIT：交易订单被交易投诉冻结<br>- DEPARTURE\_AMOUNT\_NO\_ENOUGH：剩余可出境金额不足<br>- BASIC\_AMOUNT\_NO\_ENOUGH：商户基本户余额不足<br>- PAYMENT\_NOT\_SUPPORT\_DEPARTURE：该笔订单不支持出境<br>- OUT\_ORDER\_ID\_DUPLICATE：同一个out\_order\_id用于不同的支付订单<br>- RISK\_CONTROL：订单被风控拦截<br>- SYSTEM\_ERROR：系统失败<br>- FEE\_ACCOUNT\_NOT\_OPEN：电商平台承担手续费但是未开通手续费账户或者手续费账户被处罚<br>- PAYER\_ACCOUNT\_ABNORMAL：资金出境方账户异常<br>- GOODS\_INFO\_ILLEGAL: 资金出境申请商品信息非法<br>- FOREIGN\_CURRENCY\_NOT\_SUPPORT: 不支持的币种类型，请换币种重试，目前仅支持八大币种：USD、HKD、JPY、EUR、GBP、CAD、AUD、SGD<br>- PAYEE\_INFO\_ILLEGAL：校验收款人信息失败<br>- PRESALE\_INFO\_ILLEGAL：资金出境申请预售信息非法<br>- 示例值：DEPARTURE\_AMOUNT\_NO\_ENOUGH |
| amount | integer | true | **【请求出境人民币金额】** 需要出境的人民币金额，单位：分 |
| foreign_amount | integer | false | **【真实出境外币金额】** 真实出境的外币金额，单位：该币种最小计价单位，当result为SUCCESS时有这个字段 |
| foreign_currency | string(20) | true | **【外币币种】** 出境的目标币种，由商户在资金出境申请接口传入。 |
| rate | integer | false | **【汇率】** 当result为SUCCESS时有这个字段，标价币种与支付币种的兑换比例乘以10的8次方即为此值，例如美元兑换人民币的比例为6.5，则rate=650000000 |
| exchange_rate_time | string(64) | false | **【购汇时间】** 当result为SUCCESS时有这个字段。遵循rfc3339标准格式，格式为yyyy-MM-DDTHH:mm:ss+TIMEZONE，yyyy-MM-DD表示年月日，T出现在字符串中，表示time元素的开头，HH:mm:ss表示时分秒，TIMEZONE表示时区（+08:00表示东八区时间，领先UTC 8小时，即北京时间）。例如：2015-05-20T13:29:35+08:00表示，北京时间2015年5月20日 13点29分35秒。 |
| estimate_exchange_rate_time | string(64) | false | **【预计购汇时间】** 当result为ACCEPT时可能有这个字段，以实际结果为准。<br>遵循rfc3339标准格式，格式为yyyy-MM-DDTHH:mm:ss+TIMEZONE，yyyy-MM-DD表示年月日，T出现在字符串中，表示time元素的开头，HH:mm:ss表示时分秒，TIMEZONE表示时区（+08:00表示东八区时间，领先UTC 8小时，即北京时间）。例如：2015-05-20T13:29:35+08:00表示，北京时间2015年5月20日 13点29分35秒。 |
| departure_amount | integer | false | **【真实出境人民币金额】** 真正出境的人民币金额，单位：分，如果是二级商户承担手续费且非收支分离，该金额=请求出境金额-手续费，否则该金额=请求出境金额 |
| fee | integer | false | **【手续费人民币金额】** 资金出境手续费人民币金额，单位：分 |
| charge_mchid | string(64) | false | **【手续费承担商户号】** 手续费承担商户号 |
| charge_account_type | string | false | **【手续费承担账户】** 基本账户或者手续费账户<br>可选取值：<br><ul><li>`BASIC`: 基本账户</li></ul><br><ul><li>`FEES`: 手续费账户</li></ul> |
