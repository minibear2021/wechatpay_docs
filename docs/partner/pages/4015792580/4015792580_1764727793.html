上传电子发票文件

更新时间：2025.12.03

调用【将电子发票插入微信用户卡包】接口之前，需要先调用本接口上传电子发票文件，获取文件ID。上传的发票文件会在三天后过期，因此上传后请尽快调用【将电子发票插入微信用户卡包】接口执行插入微信用户卡包的操作。

接口说明

支持商户：【普通服务商】

请求方式：【POST】/v3/new-tax-control-fapiao/fapiao-applications/upload-fapiao-file

请求域名：【主域名】https://api.mch.weixin.qq.com 使用该域名将访问就近的接入点

【备域名】https://api2.mch.weixin.qq.com 使用该域名将访问异地的接入点 ，指引点击查看

请求参数
折叠全部参数

Header  HTTP头参数

 Authorization 必填　string

请参考签名认证生成认证信息

 Accept 必填　string

请设置为application/json

 Content-Type 必填　string

请设置为multipart/form-data

Form  表单

 file 必填   file

【电子发票文件】 将电子发票文件转换为二进制数据流，在请求body中通过表单提交的方式上传此二进制内容。电子发票文件只支持PDF格式，文件大小不能超过2M

 meta 必填   object

【电子发票文件元信息】 电子发票文件元信息

属性

 sub_mchid 必填   string(32)

【子商户号】 微信支付分配的子商户号，具体请参考服务商模式开发必要参数说明

 file_type 必填   string

【文件类型】 发票文件的类型

可选取值

  • PDF:  PDF文件类型

 digest_alogrithm 必填   string

【文件摘要算法】 文件摘要算法

可选取值

  • SM3:  国密SM3摘要算法

 digest 必填   string(64)

【文件摘要】 根据文件摘要算法对电子发票文件二进制内容计算出的文件摘要，结果为16进制编码

文件上传步骤

1.创建一个POST的方法请求/upload URI:

例如： POST https://api.mch.weixin.qq.com/new-tax-control-fapiao/fapiao-applications/upload-fapiao-file  HTTP/1.1

2.将文件的数据添加到请求主体:

2.1 文本文件file参数的获取方式说明：

文本文件二进制内容，放在请求HTTP的body中。

2.2 电子发票文件元信息meta参数的获取方式说明： 电子发票文件元信息，使用json表示，包含四个对象：sub_mchid、file_type、digest_alogrithm、digest。

  • sub_mchid参数获取方式说明： 微信支付分配的子商户号，服务商模式下必传

  • file_type参数获取方式说明： 发票文件的类型为PDF - PDF文件类型，PDF文件类型

  • digest_alogrithm参数获取方式说明： 文件摘要算法，SM3 - 国密SM3算法，国密SM3摘要算法

  • digest参数获取方式说明： 根据文件摘要算法对电子发票文件二进制内容计算出的文件摘要，结果为16进制编码

2.3 签名计算说明：

签名生成

参与签名计算的请求主体为meta的json串：

1{ "file_type": "PDF","digest": "addca90a6a290b9642dbaccffdf01e8c22aa348940b2e96754169ba08c19e5db", "digest_alogrithm": "SM3" }
