1. 如存在自费支付金额，通过自费下单接口获取到发起自费支付的必要参数timeStamp、nonceStr、package、signType、paySign

2. 通过【医保自费混合收款下单】接口获取到发起支付的必要参数mix\_trade\_no

3. 使用wx.requestMedicalInsurancePay方法调起医保自费混合支付

4. 用户输入密码，确认收款

5. 返回医疗机构小程序页面（通过小程序生命周期的onShow监听返回小程序页面的事件），医疗机构小程序调用【查看医保自费混合收款结果】接口，查询收款结果，更新小程序页面


## 接口说明

支持商户： 【间连商户（医疗机构）】

接口名称： wx.requestMedicalInsurancePay

调用wx.requestMedicalInsurancePay(OBJECT)发起微信支付

## 接口兼容

iOS兼容性表现：若微信版本>=8.0.44，开发者可以通过此接口调起医保自费混合支付；若微信版本<8.0.44，开发者不能通过此接口调起医保自费混合支付，此时开发者应提示用户更新微信版本。

Android兼容性表现：若微信版本>=8.0.44，开发者可以通过此接口调起医保自费混合支付；若微信版本<8.0.44，开发者不能通过此接口调起医保自费混合支付，此时开发者应提示用户更新微信版本。

鸿蒙兼容性表现：若微信版本>=8.0.13，开发者可以通过此接口调起医保自费混合支付；若微信版本<8.0.13，开发者不能通过此接口调起医保自费混合支付，此时开发者应提示用户更新微信版本。

## 字段说明

| 参数名 | 类型 | 必填 | 描述 |
| --- |  --- | --- | --- |
| mixTradeNo | string(256) | true | 混合收款单ID |
| timeStamp | string(32) | false | 时间戳，标准北京时间，时区为东八区，自1970年1月1日 0点0分0秒以来的秒数。注意：部分系统取到的值为毫秒级，需要转换成秒(10位数字)。有自费必填。 |
| nonceStr | string(32) | false | 随机字符串，不长于32位。有自费必填。 |
| package | string(128) | false | JSAPI下单接口返回的prepay\_id参数值，提交格式如：prepay\_id=\*\*\*。有自费必填。 |
| signType | string(32) | false | 签名类型，默认为RSA，仅支持RSA。有自费必填。 |
| paySign | string(256) | false | 签名，使用字段AppID、timeStamp、nonceStr、package计算得出的签名值。有自费必填。 |
