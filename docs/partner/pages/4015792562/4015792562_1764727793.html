创建电子发票卡券模板

更新时间：2025.12.03

为商户创建电子发票的卡券模板，该卡券模板在开具的电子发票插入微信用户卡包时使用。调用该接口将覆盖商户之前配置的电子发票卡券模板。
注意：商户必须先调用本接口创建卡券模版之后才能调用开票相关接口。

接口说明

支持商户：【普通服务商】

请求方式：【POST】/v3/new-tax-control-fapiao/card-template

请求域名：【主域名】https://api.mch.weixin.qq.com 使用该域名将访问就近的接入点

【备域名】https://api2.mch.weixin.qq.com 使用该域名将访问异地的接入点 ，指引点击查看

请求参数
折叠全部参数

Header  HTTP头参数

 Authorization 必填　string

请参考签名认证生成认证信息

 Accept 必填　string

请设置为application/json

 Content-Type 必填　string

请设置为application/json

body  包体参数

 sub_mchid 必填   string(32)

【子商户号】 微信支付分配的子商户号，具体请参考服务商模式开发必要参数说明

 card_appid 必填   string(32)

【插卡服务号AppID】 插卡服务号AppID。AppID需要与服务商或子商户商户号绑定，具体请参考服务商模式开发必要参数说明

 card_template_information 必填   object

【卡券模板信息】 卡券模板信息

属性

 payee_name 选填   string(32)

【收款方名称】 收款方名称。显示在电子发票卡券信息中，若不传则默认取商户名称

 logo_url 必填   string(1024)

【卡券logo地址】 卡券logo地址。使用插卡服务号身份调用【上传永久素材】接口上传logo图片，得到的图片素材url地址

 custom_cell 选填   object

【卡券自定义cell位配置】 卡券自定义cell位配置。需要在卡券详情中增加

属性

 words 必填   string(5)

【cell位文字】 展示在卡券详情页自定义cell位上的文字

 description 必填   string(6)

【cell位描述】 展示在卡券详情页自定义cell位右侧的描述

 jump_url 选填   string(1024)

【点击cell位跳转的地址】 点击卡券详情页自定义cell位后跳转的页面链接，用户跳转时会自动在链接后加上参数encrypt_code与card_id，商户需要调用code解码接口解码encrypt_code得到真实的code

 miniprogram_user_name 选填   string(32)

【点击cell位跳转的小程序的用户名】 用于指定点击卡券详情页自定义cell位后跳转的小程序，该小程序必须与商户插卡的AppID绑定

 miniprogram_path 选填   string(1024)

【点击cell位跳转的小程序的页面路径】 用于指定点击卡券详情页自定义cell位后跳转的小程序页面路径，跳转小程序后，开发者可以通过页面onShow方法的query中获取到发起跳转的card_id、encrypt_code和用户的公众号OpenID，商户需要调用code解码接口解码encrypt_code得到真实的code
