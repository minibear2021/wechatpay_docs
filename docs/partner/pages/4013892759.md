电商平台发起申请后, 电商平台调用此接口查询审批和出款进度




## 接口说明

支持商户：【平台商户】

请求方式：【GET】 `/v3/ecommerce/account/apply-cancel-withdraw/out-request-no/{out_request_no}`

请求域名：【主域名】https://api.mch.weixin.qq.com 使用该域名将访问就近的接入点

　　　　　【备域名】https://api2.mch.weixin.qq.com 使用该域名将访问异地的接入点，指引[点击查看](https://pay.weixin.qq.com/doc/v3/partner/4012082567.md)

| 参数名 | 类型 | 必填 | 描述 |

## 应答参数

200 OK

| 参数名 | 类型 | 必填 | 描述 |
| --- |  --- | --- | --- |
| applyment_id | string(32) | true | 【微信支付注销提现申请单号】 电商平台提交注销申请后，由微信支付返回的申请单号，作为查询申请状态的唯一标识。 |
| out_request_no | string(32) | true | 【商户注销申请单号】 商户注销申请单号，由商户自定义生成，需要全局唯一 |
| cancel_state | string | true | 【注销提现申请单状态】 发起注销提现申请后，整个流程可能流转的状态<br>可选取值<br><ul><li>`ACCEPTED`:  已受理</li><li>`REVIEWING`:  审核中</li><li>`REJECTED`:  审批驳回</li><li>`WAITING_MERCHANT_CONFIRM`:  等待商户确认</li><li>`REVOKED`:  已撤销</li><li>`SYSTEM_PROCESSING`:  系统处理中</li><li>`CANCELED`:  已注销</li><li>`FUND_PROCESSING`:  资金处理中</li><li>`FINISH`:  注销完成</li></ul> |
| cancel_state_description | string(256) | true | 【注销提现申请单状态描述】 注销提现申请单状态的文字描述 |
| withdraw | string | false | 【是否提取资金】 在提交接口传入的是否提取资金标记<br>可选取值<br><ul><li>`NOT_APPLY_WITHDRAW`:  仅注销，不申请提现</li><li>`APPLY_WITHDRAW`:  申请提现</li></ul> |
| withdraw_state | string | false | 【提现状态】 当申请单进入「资金处理中」状态时，返回此状态，反映资金提现的处理进度<br>可选取值<br><ul><li>`WITHDRAW_PROCESSING`:  提现处理中</li><li>`WITHDRAW_EXCEPTION`:  提现异常</li><li>`WITHDRAW_SUCCEED`:  提现成功</li></ul> |
| withdraw_state_description | string(256) | false | 【提现状态描述】 提现申请单状态的文字描述 |
| account_withdraw_result | array[object] | false | 【账户提现结果】 对具体账户发起提现操作后返回，商户已开通的账户参见 `account_info` 字段返回，微信支付注销提现流程会将商户账户里所有遗留的资金全额清退，请平台做好资金对账。多个账户存在资金时，将会有多笔提现金额，请注意核对银行入账流水 |
| account_withdraw_result.out_account_type | string | true | 【二级商户号的出款子账户类型】  仅返回已开通的账户类型<br>可选取值<br><ul><li>`BASIC_ACCOUNT`:  商户的基本户</li><li>`OPERATE_ACCOUNT`:  商户的运营账户</li><li>`MARGIN_ACCOUNT`:  商户的保证金户</li><li>`TRADE_FEE_ACCOUNT`:  商户的手续费账户</li></ul> |
| account_withdraw_result.pay_state | string | true | 【付款状态】 付款状态<br>可选取值<br><ul><li>`PAY_PROCESSING`:  处理中</li><li>`PAY_SUCCEED`:  付款成功</li><li>`PAY_FAIL`:  付款失败</li><li>`BANK_REFUNDED`:  银行退票</li></ul> |
| account_withdraw_result.state_description | string(256) | true | 【付款状态描述】 付款状态描述 |
| modify_time | string(32) | false | 【最后更新时间】 遵循 [rfc3339](https://datatracker.ietf.org/doc/html/rfc3339) 标准格式，格式为yyyy-MM-DDTHH:mm:ss+TIMEZONE，yyyy-MM-DD表示年月日，T出现在字符串中，表示time元素的开头，HH:mm:ss表示时分秒，TIMEZONE表示时区（+08:00表示东八区时间，领先UTC 8小时，即北京时间）。例如：2015-05-20T13:29:35+08:00表示，北京时间2015年5月20日 13点29分35秒。 |
| sub_mchid | string(32) | true | 【申请注销的二级商户号】 平台商户的二级商户号，由微信支付生成并下发 |
| account_info | array[object] | false | 【账户信息】二级商户号已开通的账户信息，仅涉及提取资金类的申请单返回，单据受理及后续状态返回 |
| account_info.out_account_type | string | true | 【二级商户号的出款子账户类型】 仅返回已开通的账户类型<br>可选取值<br><ul><li>`BASIC_ACCOUNT`:  商户的基本户</li><li>`OPERATE_ACCOUNT`:  商户的运营账户</li><li>`MARGIN_ACCOUNT`:  商户的保证金户</li><li>`TRADE_FEE_ACCOUNT`:  商户的手续费账户</li></ul> |
| account_info.amount | integer | true | 【账户金额】 单位：分 |
| confirm_cancel | object | false | 【用于引导商户超级管理员确认注销的信息】 当注销提现申请单状态变更为WAITING\_MERCHANT\_CONFIRM（等待商户确认）时，查单接口会返回此字段组。此时可将对应url转化为二维码，引导商户号员工扫码确认注销。 |
| confirm_cancel.confirm_cancel_url | string(1024) | false | 【确认商户注销的页面url】 确认商户注销的页面url，推荐：把url转为二维码，邀请被注销商户的员工使用微信扫描二维码访问，确认注销商户。 |
