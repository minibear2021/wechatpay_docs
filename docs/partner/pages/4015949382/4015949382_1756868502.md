### 2.2、处理商户平台处置记录回调通知

回调地址设置成功后，确保回调地址可以正常访问的情况下，子商户违规处理记录和交易拦截记录时，可以接收到对应的推送

```
import okhttp3.Headers;

// 商户APIv3密钥 https://pay.weixin.qq.com/doc/v3/partner/4013059095
String apiv3Key = "xxxxxxx";
// 微信支付公钥ID，如何获取请参考 https://pay.weixin.qq.com/doc/v3/partner/4013038589
String wechatPayPublicKeyId = "PUB_KEY_ID_xxxxxxxxxxxxx";
// 微信支付公钥文件路径，本地文件路径
String wechatPayPublicKeyFilePath = "/path/to/wxp_pub.pem";
// 此处为代码示例，获取微信支付通知 Header 列表
Headers headers = getHeaders();
// 此处为代码示例，获取微信支付通知 Body
String body = getBody();

PublicKey wechatPayPublicKey WXPayUtility.loadPublicKeyFromPath(wechatPayPublicKeyFilePath);

Notification notification;
try {
  // 1. 解析微信支付回调包体，body为商户接收到微信回调的原始信息
  notification  = WXPayUtility.parseNotification(apiv3Key, wechatPayPublicKeyId, wechatPayPublicKey, headers, body);
  // 2. 解析完回调包体之后，可以先返回给微信支付200或者204，表示回调成功
  //    商户再异步处理包体逻辑，这样子能保证微信回调不会超时，微信支付超时多次之后不会再回调
  send2MQ(notification);
} catch (Exception e) {
  // 处理回调异常
}
```
