接口适用场景：当服务商提交申请单后，需要定期调用此接口查询申请单的审核状态。




## 接口说明

支持商户：【平台商户】

请求方式：【GET】 `/v3/ecommerce/applyments/out-request-no/{out_request_no}`

请求域名：【主域名】https://api.mch.weixin.qq.com 使用该域名将访问就近的接入点

　　　　　【备域名】https://api2.mch.weixin.qq.com 使用该域名将访问异地的接入点，指引[点击查看](https://pay.weixin.qq.com/doc/v3/partner/4012082567.md)

## 请求参数

### Header 参数
| 参数名 | 类型 | 必填 | 描述 |
| --- |  --- | --- | --- |
| Authorization | string | true | 请参考 [签名认证](https://pay.weixin.qq.com/doc/v3/partner/4012365870.md#%E8%AF%B7%E6%B1%82%E6%8E%A5%E5%8F%A3%E5%90%8E%E7%AB%AF%E7%AD%BE%E5%90%8D) 生成认证信息 |
| Accept | string | true | 请设置为 `application/json` |
### Path 参数
| 参数名 | 类型 | 必填 | 描述 |
| --- |  --- | --- | --- |
| out_request_no | string(124) | true | **【业务申请编号】** 1、服务商自定义的商户唯一编号。<br>2、每个编号对应一个申请单，每个申请单审核通过后会生成一个微信支付商户号。 |
## 应答参数

200 OK

| 参数名 | 类型 | 必填 | 描述 |
| --- |  --- | --- | --- |
| applyment_state | string(32) | true | **【申请状态】** 枚举值：<br>CHECKING：资料校验中<br>ACCOUNT\_NEED\_VERIFY：待账户验证<br>AUDITING：审核中<br>REJECTED：已驳回<br>NEED\_SIGN：待签约<br>FINISH：完成<br>FROZEN：已冻结<br>CANCELED：已作废 |
| applyment_state_desc | string(1024) | true | **【申请状态描述】** 申请状态描述, 示例："审核中" |
| sign_url | string(256) | false | **【签约链接】** 1、当申请状态为NEED\_SIGN时才返回。<br>2、建议将链接转为二维码展示，需让申请单-管理者用微信扫码打开，完成签约。 |
| sub_mchid | string(32) | false | **【电商平台二级商户号】** 当申请状态为NEED\_SIGN或FINISH时才返回。 |
| account_validation | object | false | **【汇款账户验证信息】** 当申请状态为ACCOUNT\_NEED\_VERIFY 时有返回，可根据指引汇款，完成账户验证 |
| account_validation.account_name | string(1024) | true | **【付款户名】** 需商户使用该户名的账户进行汇款。<br>该字段需进行解密处理，解密方法详见 [如何使用API证书解密敏感字段](https://pay.weixin.qq.com/doc/v3/partner/4013059060.md)。 |
| account_validation.account_no | string(1024) | false | **【付款卡号】** 结算账户为对私时会返回，商户需使用该付款卡号进行汇款。<br>该字段需进行解密处理，解密方法详见 [如何使用API证书解密敏感字段](https://pay.weixin.qq.com/doc/v3/partner/4013059060.md)。 |
| account_validation.pay_amount | integer | true | **【汇款金额】** 需要汇款的金额(单位：分)。 |
| account_validation.destination_account_number | string(128) | true | **【收款卡号】** 收款账户的卡号 |
| account_validation.destination_account_name | string(128) | true | **【收款户名】** 收款账户名 |
| account_validation.destination_account_bank | string(128) | true | **【开户银行】** 收款账户的开户银行名称。 |
| account_validation.city | string(128) | true | **【省市信息】** 收款账户的省市。 |
| account_validation.remark | string(128) | true | **【备注信息】** 商户汇款时，需要填写的备注信息。 |
| account_validation.deadline | string(128) | true | **【汇款截止时间】** 请在此时间前完成汇款。 |
| audit_detail | array[object] | false | **【驳回原因详情】** 各项资料的审核情况。当申请状态为REJECTED或 FROZEN时才返回。 |
| audit_detail.param_name | string(32) | true | **【参数名称】** 提交申请单的资料项名称。 |
| audit_detail.reject_reason | string(1024) | true | **【驳回原因】** 提交资料项被驳回原因。 |
| legal_validation_url | string(256) | false | **【法人验证链接】** 1、当申请状态为<br>ACCOUNT\_NEED\_VERIFY，且通过系统校验的申请单，将返回链接。<br>2、建议将链接转为二维码展示，让商户法人用微信扫码打开，完成账户验证。 |
| out_request_no | string(128) | true | **【业务申请编号】** 提交接口填写的业务申请编号 |
| applyment_id | integer | true | **【微信支付申请单号】** 微信支付分配的申请单号 |
| sign_state | string(32) | false | **【签约状态】** 1、未签约-UNSIGNED：该状态下，电商平台可查询获取签约链接，引导二级商户的超级管理员完成签约；<br>2、已签约-SIGNED ：指二级商户的超级管理员已完成签约。注意：若申请单被驳回，商户修改了商户主体名称、法人名称、超级管理员信息、主体类型等信息，则需重新签约。<br>3、不可签约-NOT\_SIGNABLE：该状态下，暂不支持超级管理员签约。一般为申请单处于已驳回、已冻结、机器校验中状态，无法签约。 |
## 错误码

### 公共错误码

| 状态码 | 错误码 | 描述 | 解决方案 |
| --- | --- | --- | --- |
| 400 | PARAM\_ERROR | 参数错误 | 请根据错误提示正确传入参数 |
| 400 | INVALID\_REQUEST | HTTP 请求不符合微信支付 APIv3 接口规则 | 请参阅 [接口规则](https://pay.weixin.qq.com/doc/v3/partner/4012081726.md) |
| 401 | SIGN\_ERROR | 验证不通过 | 请参阅 [签名常见问题](https://pay.weixin.qq.com/doc/v3/partner/4012082324.md) |
| 500 | SYSTEM\_ERROR | 系统异常，请稍后重试 | 请稍后重试 |

### 业务错误码

| 状态码 | 错误码 | 描述 | 解决方案 |
| --- | --- | --- | --- |
| 400 | RESOURCE\_ALREADY\_EXISTS | 存在流程进行中的申请单，请检查是否重入 | 可通过查询申请状态查看此申请单的申请状态，或更换out\_request\_no提交新的申请单 |
| 403 | NO\_AUTH | 商户权限异常 | 请确认是否已经开通相关权限 |
| 404 | RESOURCE\_NOT\_EXISTS | 申请单不存在 | 确认入参，传入正确的申请单编号 |
| 500 | RATELIMIT\_EXCEEDED | 系统错误 | 系统异常，请使用相同参数稍后重新调用 |
