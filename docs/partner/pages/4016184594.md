更新投放计划




## 接口说明

支持商户：【普通服务商】

请求方式：【PATCH】 `/v3/marketing/partner/delivery-plan/delivery-plans/{plan_id}`

请求域名：【主域名】https://api.mch.weixin.qq.com 使用该域名将访问就近的接入点

　　　　　【备域名】https://api2.mch.weixin.qq.com 使用该域名将访问异地的接入点，指引[点击查看](https://pay.weixin.qq.com/doc/v3/merchant/4012075113.md)

接口限频：100/秒（商户号维度）

## 请求参数

| --- |  --- | --- | --- |
| Authorization | string | true | 请参考 [签名认证](https://pay.weixin.qq.com/doc/v3/merchant/4012365342.md#%E8%AF%B7%E6%B1%82%E6%8E%A5%E5%8F%A3%E5%90%8E%E7%AB%AF%E7%AD%BE%E5%90%8D) 生成认证信息 |

| Content-Type | string | true | 请设置为 `application/json` |
### Path 参数
| 参数名 | 类型 | 必填 | 描述 |
| --- |  --- | --- | --- |
| plan_id | string(32) | true | 【投放计划ID】 创建生成的投放计划 ID |
### Body 参数
| 参数名 | 类型 | 必填 | 描述 |
| --- |  --- | --- | --- |
| modify_content | object | true | 【修改内容】 修改内容，修改哪个字段就填入哪个字段 |
| modify_content.plan_name | string(36) | false | 【投放计划名称】 投放计划，用于内部管理，不做C端展示 |
| modify_content.delivery_end_time | string | false | 【投放结束时间】 投放结束时间，遵循 [rfc3339](https://datatracker.ietf.org/doc/html/rfc3339) 标准格式，格式为YYYY-MM-DDTHH:mm:ss+TIMEZONE，YYYY-MM-DD表示年月日，T出现在字符串中，表示time元素的开头，HH:mm:ss表示时分秒，TIMEZONE表示时区（+08:00表示东八区时间，领先UTC 8小时，即北京时间）。例如：2015-05-20T13:29:35+08:00表示，北京时间2015年5月20日 13点29分35秒。注：活动结束时间仅支持延长，不支持减少。商品券会随批次结束时间到期而直接失效，因此投放活动的结束时间需早于批次时间，保证用户不会在活动期间领取到一张立即过期的券。<br>情况1: 若商品券配置了wait\_days\_after\_receive（领券后需等待N天生效）和available\_days（券生效后N天内可用），则活动结束时间最长可配置为【available\_end\_time（批次结束时间）- wait\_days\_after\_receive（领券后需等待N天生效）- available\_days（券生效后N天内可用） -1 】。<br>情况2: 若商品券没有配置了wait\_days\_after\_receive（领券后需等待N天生效）和available\_days（券生效后N天内可用），则活动结束时间最长可配置为【available\_end\_time（批次结束时间） -1】。 |
| modify_content.total_count | integer | false | 【投放总库存数量】 约定了投放计划的总投放库存，创建投放计划的券可用库存需在 10000 以上。注：修改后活动库存需大于原库存，不支持减少库存。 |
| modify_content.user_limit | integer | false | 【单用户限领】 约定了投放计划用户维度的限领；非必填，如创建时未填写，则修改时不支持填写。注：修改后单用户限额需大于原单用户限额，不支持减少限额。 |
| modify_content.daily_limit | integer | false | 【单日限领】 约定了投放计划单用户单日维度的限领；非必填，如创建时未填写，则修改时不支持填写。注：修改后活动日限额需大于原限额，不支持减少日限额。 |
| modify_content.recommend_word | string(27) | false | 【营销标签】 用于在优惠左上角展示的运营推荐语信息。自定义文案，不超过9个中文字符或18个英文字符 |
| out_request_no | string(40) | true | 【修改请求单号】 修改投放计划的请求流水号，品牌侧需保持唯一性，可使用 数字、大小写字母、下划线 `_`、短横线 `-` 组成，长度在6-40个字符之间 |
