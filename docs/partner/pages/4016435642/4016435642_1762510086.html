开发指引

更新时间：2025.11.07

1、背景介绍

为解决在摇一摇有优惠场景“由于用户在商家小程序未登录正确的领券账号，导致无法找到优惠券”和“用户领取优惠券后在商家小程序未能找到优惠券适用的商品”这两大核心问题，平台侧提供标准化核销组件，用于提升用户领券率与领取核销率。完成核销组件开发后，需对摇优惠活动重新进行测试，详细指引请参考：品牌经营平台管理活动。

2、组件介绍

组件页面无需商家开发，商家根据规范在用户用券场景调起核销组件，串接可用单品。

 

3、如何开通商品券核销组件权限

需联系贵司对接的微信支付行业运营负责人申请。

4、系统版本要求

开发者可通过调用wx.openBusinessView方式在商户小程序中打开商品券核销组件，引导用户核销商品券。

接口兼容：微信 >= 8.0.30&小程序版本库 >= 2.28.1，低版本需提示用户升级到最新的微信版本。

step1、从摇一摇领券事件回调通知获取用户openid

step2、判断用户登录账号状态

商家侧操作说明：

  1. 参数校验：从路径 query 参数获取 couponCode（券的唯一标识)

  2. 身份验证：

    1. 商家使用 couponCode 和 stockId 匹配用户领券openid。

    2. 通过商家服务端接口获得当前用户小程序登陆openid。

    3. 比对 领券 openid 和 登录 openid 的一致性，按照协议设置 authState(用户账户登录状态，见本文档5.3 商家调起商品券组件接口说明，extraData 参数 authState 的说明 )

step3、商家调起核销组件接口

开发者可通过调用 wx.openBusinessView 方式在本小程序打开商品券核销组件，引导用户核销商品券。

接口名称：wx.openBusinessView

接口兼容：微信 >= 8.0.30&小程序版本库 >= 2.28.1，低版本需提示用户升级到最新的微信版本。
