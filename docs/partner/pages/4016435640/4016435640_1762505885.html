调起小程序核销组件

更新时间：2025.11.07

开发者可通过调用wx.openBusinessView方式在商户小程序打开摇优惠核销组件，引导用户核销商品券。

 

版本更新说明

日期

改动说明

2025-04-22

新增全场券点击事件selectGlobalCoupon

接口说明

支持商户： 【普通服务商】

接口名称：wx.openBusinessView

接口兼容：微信 >= 8.0.30&小程序版本库 >= 2.28.1，低版本需提示用户升级到最新的微信版本。

字段说明

请求参数

businessType   必填  string

固定配置：wxpayCouponUse

extraData   必填  Object

用于向组件传递商品券信息

属性

couponCode   必填  string

【券的唯一标识】商品详情页跳转商家小程序的路径query会增加该属性，开发者可通过小程序页面query参数获取此值。

stockId   必填  string

【批次号】商品详情页跳转商家小程序的路径query会增加该属性，开发者可通过小程序页面query参数获取此值。

authState   必填  string

【用户登录账号的状态】

verified：用户正确登录账号

invalidCredentials：用户登录错误账号

unauthenticated：用户未登录/注册

具体可参考文档开发指引

wx.openBusinessView 请求示例

1if (wx.openBusinessView && checkVersion()) {
2//checkVersion版本检查：微信>=8.0.30&小程序版本库 >= 2.28.1
3  wx.openBusinessView({
4    businessType: 'wxpayCouponUse',
5    extraData: {
6      couponCode: '1227944959000000911017', // 从query获取
7      stockId: '1286950000000039',   // 从query获取
8      authState: 'verified' // 见本文档摇优惠-核销组件（通过openid发券）开发指引 - step2、判断用户登录账号状态
9    },
10    success(e) {
11        console.log(e.extraData.action); // 用户动作
12        console.log(e.extraData.productPath); // 用户选择的商品路径
13      // dosomething
14    },
15    fail() {
16      // dosomething
17    },
18    complete() {
19      // dosomething
20    }
21  });
22} else {
23  // 引导用户升级微信版本
24}

返回参数

success回调结果参数

extraData   必填  Object

用于向组件传递商品券信息

属性

action   选填  string

【用户动作】

selectProduct: 选择商品

selectGlobalCoupon：选择全场券

auth: 注册/登录

closePage: 关闭页面

switchAccount: 更换登录账号

productPath   选填  string

【用户选择的商品路径】当用户选择商品，会在success回调返回商品路径，由开发者自行跳转

errMsg   选填  string

【错误信息】

 

小程序生命周期说明

  1. 商户小程序打开摇优惠核券组件时，会触发 App onShow 和 Page onShow 事件

  2. 当摇优惠核券组件关闭时，会触发 App onHide 和 Page onHide 事件
