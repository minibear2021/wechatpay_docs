通过时间范围，邀请的code等参数查询服务商邀请的商户信息，支持分页查询。




## 接口说明

支持商户：【普通服务商】

请求方式：【GET】 `/v3/new-tax-control-fapiao/fapiaomerchant/listspinvitemchinfo`

请求域名：【主域名】https://api.mch.weixin.qq.com 使用该域名将访问就近的接入点

　　　　　【备域名】https://api2.mch.weixin.qq.com 使用该域名将访问异地的接入点，指引[点击查看](https://pay.weixin.qq.com/doc/v3/merchant/4012075113.md)

## 请求参数

### Header 参数
| 参数名 | 类型 | 必填 | 描述 |
| --- |  --- | --- | --- |
| Authorization | string | true | 请参考 [签名认证](https://pay.weixin.qq.com/doc/v3/merchant/4012365342.md#%E8%AF%B7%E6%B1%82%E6%8E%A5%E5%8F%A3%E5%90%8E%E7%AB%AF%E7%AD%BE%E5%90%8D) 生成认证信息 |
| Accept | string | true | 请设置为 `application/json` |
### Query 参数
| 参数名 | 类型 | 必填 | 描述 |
| --- |  --- | --- | --- |
| query_time_start | string | true | 【查询时间起】 查询邀请商户的起始时间，遵循 [RFC3339](https://tools.ietf.org/html/rfc3339) 标准格式：yyyy-MM-DDTHH:mm:ss+TIMEZONE |
| query_time_end | string | true | 【查询时间止】 查询商户邀请的结束时间，与query\_time\_start的间隔时间不能大于7天。遵循 [RFC3339](https://tools.ietf.org/html/rfc3339) 标准格式：yyyy-MM-DDTHH:mm:ss+TIMEZONE |
| offset | integer | true | 【本次查询的起始位置】 当前查询的分页页数。从0开始计数，如offset=10，表示从第11条记录开始返回 |
| limit | integer | true | 【本次查询的最大数量】 每页所查询的商户信息数量 |
| invite_code | string | false | 【服务商邀请code】 邀请链接唯一标识 |
| mch_invite_status | string | true | 【商户邀请状态】 商户邀请状态<br>可选取值<br><ul><li>`MCH_INVITE_SUCC`:  商户已接受邀请，通过 [【检查子商户开票功能状态】](https://pay.weixin.qq.com/doc/v3/partner/4015792561.md) 接口查询具体状态</li><li>`MCH_INVITE_FAILED`:  商户邀请开通失败，具体失败原因查看invite\_failed\_reason字段</li></ul> |
## 应答参数

200 OK

| 参数名 | 类型 | 必填 | 描述 |
| --- |  --- | --- | --- |
| total_count | integer | true | 【总数】 满足查询条件的所有邀请商户总数 |
| offset | integer | true | 【本次查询的起始位置】 本次查询的起始位置。从0开始计数，如offset=10，表示从第11条记录开始返回 |
| limit | integer | true | 【本次查询的最大数量】 每页所查询的商户信息数量 |
| links | object | false | 【相关链接】 相关链接 |
| links.next | string(1024) | false | 【下一页链接】 使用同样的limit进行下一页查询时的请求链接。当前为最后一页时，返回为空 |
| links.prev | string(1024) | false | 【上一页链接】 使用同样的limit进行上一页查询时的请求链接。如果为第一页时，返回为空 |
| links.self | string(1024) | false | 【当前链接】 当前的请求链接 |
| mch_invite_result_list | array[object] | false | 【邀请的商户结果】 服务商邀请的商户结果列表 |
| mch_invite_result_list.sub_mchid | string | false | 【邀请的商户商户号】 服务商邀请的商户号，当邀请状态为MCH\_INVITE\_SUCC返回 |
| mch_invite_result_list.mch_invite_status | string | true | 【邀请商户的状态】 邀请商户的具体状态<br>可选取值<br><ul><li>`MCH_INVITE_SUCC`:  商户已接受邀请，通过 [【检查子商户开票功能状态】](https://pay.weixin.qq.com/doc/v3/partner/4015792561.md) 接口查询具体状态</li><li>`MCH_INVITE_FAILED`:  商户邀请开通失败，具体失败原因查看invite\_failed\_reason字段</li></ul> |
| mch_invite_result_list.ep_name | string(1024) | true | 【商户名称】 商户名称 |
| mch_invite_result_list.tax_id | string(32) | true | 【商户税号】 商户税号 |
| mch_invite_result_list.invite_code | string | true | 【服务商邀请商户code】 邀请链接唯一标识 |
| mch_invite_result_list.operate_time | string | true | 【商户操作时间】 如果邀请状态为MCH\_INVITE\_SUCC，则是商户确认授权的时间；如果邀请状态为MCH\_INVITE\_FAILED，则是商户邀请失败的时间。遵循 [RFC3339](https://tools.ietf.org/html/rfc3339) 标准格式：yyyy-MM-DDTHH:mm:ss+TIMEZONE |
| mch_invite_result_list.invite_failed_code | string | false | 【商户邀请失败错误码】 邀请失败时返回<br>可选取值<br><ul><li>`UNKNOWN_TAX_INFO`:  未查询到单位税务登记信息，请联系税局确认税务登记信息</li><li>`INVALID_COMPANY_INFO`:  单位信息有误，请确认微信支付平台和税务局登记主体信息一致，微信支付更新主体信息参考： [商家助手修改商户法人、经营者](https://kf.qq.com/faq/230818qqIJnE230818i2mqi6.html)</li><li>`NOT_MEET_ACCESS_CONDITION`:  不符合乐企接入条件</li><li>`NOT_IN_PILOT_SCOPE`:  未纳入数电票试点范围，请参考 [如何进行发票票种核定](https://mp.weixin.qq.com/s/xjT7b0J4yAKSLPs9pYVlYA) 开通数电发票</li><li>`MISSING_TAX_AUTHORITY_CODE`:  无法获取省市级主管税务机关代码</li><li>`MERCHANT_INFO_NOT_SET`:  商户主体信息未设置，请参考 [商家助手修改商户执照、登记证书](https://kf.qq.com/faq/2308182qaMza230818mi6ZNB.html) 修改商户主体信息</li><li>`UNKNOWN_ERROR`:  未知异常</li></ul> |
| mch_invite_result_list.invite_failed_reason | string(1024) | false | 【商户邀请失败的原因】 邀请失败时返回，描述商户邀请失败的具体原因 |
