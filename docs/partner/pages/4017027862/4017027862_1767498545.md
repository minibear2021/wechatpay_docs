### Q：服务商调用“根据业务申请编号查询添加申请状态”返回申请状态为生效中，但交易链路仍未生效

A：请参照以下几点检查：

- “生效中”指申请单的流程已结束，并不能代表配置连接规则的实时状态，服务商需调用“[查询已生效交易连接名片规则](https://pay.weixin.qq.com/doc/v3/partner/4016366785.md)”接口，查看配置账号是否存在。

- 连接规则配置成功后，由于交易链路存在缓存的原因并不会立即生效，需要等待6个小时左右。


### Q：服务商调用“解除已生效交易连接名片场景”成功后，交易链路仍未失效

A：连接规则配置解除后，由于交易链路存在缓存的原因并不会立即失效，需要等待6个小时左右。

### Q：服务商调用“添加交易连接名片规则申请”返回：当前连接规则已在添加流程中，或返回：连接关系已被当前品牌或其他品牌绑定

A：目前要求一个主账号只能绑定一个品牌，为防止连接规则被重复配置，提交申请时会查询是否存在相同配置的连接规则已在添加流程中，以及是否存在相同配置的连接规则已经生效，交易场景类型不同，判断标准也不同，具体如下表所示：

| 交易场景类型 | 细分类别 | 主账号 | 绑定子账号 | 判断唯一性 |
| :-- | :-- | :-- | :-- | :-- |
| 小程序支付 | 普通小程序 | `appid` | 无 | `appid` |
| 收银类小程序 | `card_link_mchid` | `appid` | `card_link_mchid` \+ `appid` |
| APP支付 | 无 | `appid` | 无 | `appid` |
| 支付分支付 | 无 | `service_id` | 无 | `service_id` |
| 刷卡支付 | 无 | `card_link_mchid` | 无 | `card_link_mchid` |

### Q：调用“添加交易连接名片规则申请”接口后，在审核期间，为什么品牌经营平台上看不到对应的在审核中状态的申请单记录？

A：API接口的数据与web端的数据存在隔离，因此平台上看不到接口提交的处于审核中的申请单。
