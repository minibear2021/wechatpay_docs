## 二、运营规范

### 2.1、审核时效

| 提交时间 | 审核时效 |
| :-- | :-- |
| 工作日14:00前提交 | 一般可在当天审核完成 |
| 工作日14:00后提交 | 一般可在T+1个工作日审核完成 |

### 2.2、具体规范

商家在配置商品券时，应当针对参与平台优惠活动的相关商品与活动，开展领券、核销全链路体验核验工作，以保障商品及活动信息的准确性与一致性，同时确保核销环节的操作无误与体验良好。

#### 2.2.1、名称及活动说明类

商家侧券名称：

商家侧（小程序/app等）优惠券名称展示「摇优惠」或「摇一摇有优惠」关键词，并注意以下事项：

- 可在商家体系内的券名称中体现；

- 可在商家体系内的图片上进行标识（角标、Tag等）；

- 不得出现涉毒、涉暴、涉黄等违反法律法规、政策或监管部门要求或平台规定的敏感内容；

微信支付侧名称：

- 命名需覆盖「投放计划名称」「商品券名称」「商品券批次 \- 券使用说明」三类内容，可参考 “品牌 \+ SKU 系列 / 品类 + 券类型（如满减券、折扣券）” 格式，且避免出现混淆用户认知的内容。

- “封顶”“限额”“限时”“使用门槛” 等使用限制，需明确说明，无需重复体现，兼顾规范与简洁；

  若优惠券存在以下使用限制，需在「投放计划名称&商品券名称」或「商品券图片」中明确标注：

  - 最高限额 / 封顶值（如 “5折优惠券（最高抵扣 50 元）”）；

  - 限时使用（如 “满 100 减 20（工作日限时）”）；

  - 0 元券（需注明使用门槛，如 “0 元购（满 30 元可用）”）。
- 不得出现涉毒、涉暴、涉黄等违反法律法规、政策或监管部门要求或平台规定的敏感内容。

#### 2.2.2、图片类

「商品券图片」：

- 对于“封顶”“限额”“限时”“使用门槛”需要在「投放计划名称」「商品券图片」中任一项体现即可；

- 样式要求：可独立出现品牌logo图形、可出现优惠面额价格等数字信息；

- 具体规范请参考：[商品券头图设计规范](https://docs.qq.com/doc/DQ2VkYmVUYWpIY2VQ)；

- 不得出现涉毒、涉暴、涉黄等违反法律法规、政策或监管部门要求或平台规定的敏感图片。


「商品券背景图」：

与品牌logo或商品头图中元素相同/相近色的纯色图

「商品券详情图列表」：

作为活动的补充图片信息、对活动内容进行展示，需与活动直接相关

「商品券套餐组合-单品列表-单品图片」：

可独立出现品牌logo图形、可出现优惠面额价格等数字信息

#### 2.2.3、核销体验类

核销微信支付优惠券，需要确认核销体验类

跳转路径：

指微信支付侧优惠券核销跳转路径

- 非全场券，核销路径需跳转至可用券商品列表页/商品详情页；

  - 核销路径仅展示优惠商品；

  - 为保证核销体验更顺滑，核销路径上非必要不弹窗；

  - 收银台：优惠后金额无误。

### 2.3、其他须知

为充分保障用户的合法权益 ，对于集中出现消费者投诉、负面舆情等问题的商家活动，平台有权视情况暂停/下线商家的活动投放，直至商家积极处理、整改完成为止。
