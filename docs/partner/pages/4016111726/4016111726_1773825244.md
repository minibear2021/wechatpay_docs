#### Q：商圈会员积分同步接口如果传0的话，订单退款后会不会有积分动账通知？

A：不会，积分同步传0表示不积分，退款也不存在退积分，所以不会有回调。

#### Q：商圈通过交易单号进行圈店，提示：获取不到商户信息，请确认交易单号是否正确。

A：删除商圈小程序，后台关闭微信，重新打开微信加载商圈小程序，再操作圈店，即可解决该问题。

#### Q：商圈授权后前端可以获取到用户手机号吗？

A：不支持。

#### Q：如何更换或作废商圈商户号？

A：一个物理位置只能有一个商圈。更换商圈商户号需作废旧商户号，可以发邮件处理：

邮件提供商圈名称，商圈商户号，说明作废原因发送到zhsq@tencent.com

注：商圈只作废商圈边界，不会作废商圈商户号。建议不使用的商圈商户号下架相关入口和删除已圈的门店信息。

#### Q：如果一个商户既有线下商圈收款，又有线上小程序，且两者使用的收款商户号相同，用户在线上小程序的交易是否可以商圈积分？

A：可以，要求收款商户号和商圈存在圈店关系，且满足自动积分条件。

#### Q：积分插件报错：“商场会员积分功能配置错误，请联系商场工作人员（错误码： 272953870)”，什么原因？

A：请检查[商圈会员场内支付结果通知](https://pay.weixin.qq.com/doc/v3/partner/4012076414.md)应答格式是否为：http status：200，{"code":"SUCCESS","message":"成功"}，且在2s内应答。

#### Q：调用积分同步接口报错：该订单未提交积分，请检查该订单是否已成功反馈积分成功，什么原因？

A：请检查[商圈会员场内支付结果通知](https://pay.weixin.qq.com/doc/v3/partner/4012076414.md)应答格式是否为：http status：200，{"code":"SUCCESS","message":"成功"}，且在2s内应答。要求正确应答积分回调通知后，再调用[积分同步](https://pay.weixin.qq.com/doc/v3/partner/4012474133.md)接口。

#### Q：商户号被处罚了是否影响无感积分？

A：如果商户号被处罚的是收款能力，就有影响，无法积分。

#### Q：商圈用户未主动上传位置信息时，如何实现自动积分？

A：可通过以下两种方式：

1、调用[商圈会员停车状态同步](https://pay.weixin.qq.com/doc/v3/partner/4012474127.md)接口，通过同步用户停车进场/离场状态，关联其在商圈内消费行为，实现自动积分。

2、连锁品牌门店配置+下单接口回传门店ID。详细参考：[https://docs.qq.com/doc/DYnFzVFJDVnNzTVlo](https://docs.qq.com/doc/DYnFzVFJDVnNzTVlo)

#### Q：商圈圈店错误如何处理？

A：商圈管理员可以使用商户收款商户号下的微信支付订单号transaction\_id操作圈店，不需要商户通过。如果发现商圈管理员圈错，商户可以联系商圈反馈。

圈店工具使用文档：[https://docs.qq.com/doc/DYllDRXZFYld2TW12](https://docs.qq.com/doc/DYllDRXZFYld2TW12)

#### Q：针对先消费后上报位置信息的用户，是否可以在商圈开通页面将当天的交易进行补录？

A：可以，当天自动失败的订单用户可以在商圈开通页面手动点击积分。

#### Q：两个小程序appid用同一个商户号，可以圈一个商圈吗？

A：一个商圈最多可以关联5个小程序appid。

#### Q：商圈积分回调通知的openid和商圈下的openid不一致？

A：需要用户先取消授权，再重新授权。具体操作参考：

1、商圈3.0取消授权，用户在商圈小程序操作。

2、商圈1.0取消授权，用户点击该链接取消授权：[https://pay.weixin.qq.com/index.php/xphp/cmchcircle/circlebonuts/record](https://pay.weixin.qq.com/index.php/xphp/cmchcircle/circlebonuts/record)

#### Q：商圈小程序报错：“商场会员积分功能配置错误，请联系商场工作人员（错误码：269795842)”，什么原因？

A：可参考以下几点排查：

1、检查调起的商户号和APPID是否绑定；

2、检查appid和openid是否匹配，[获取openid](https://pay.weixin.qq.com/doc/v3/partner/4012081935.md#OpenID)。

#### Q：商圈小程序取消授权需要如何重新授权？

A：重新打开[插件页](https://mp.weixin.qq.com/wxopen/plugindevdoc?appid=wxfab2bf944bfc4da6&token=1853922919&lang=zh_CN)，点击立即开通。

#### Q：用户先完成交易，后续才开通无感积分，此前的订单能否补录积分？

A：不支持补录开通前的历史订单（不包括开通当天订单）。仅支持开通后，当天因不满足自动积分条件的支付订单，可补录积分。

#### Q：商圈支持圈个人用户收款码的交易记录吗？

A：不支持。

#### Q：商圈车辆入场信息的有效期是多久？

A：若未主动调用[商圈会员停车状态同步](https://pay.weixin.qq.com/doc/v3/partner/4012474127.md)接口传递state:OUT（用户开车离开商圈），该入场信息的有效期默认截止至当天23:59:59。

#### Q：商圈小程序显示积分中，什么原因？

A：需要调用[商圈会员积分同步](https://pay.weixin.qq.com/doc/v3/partner/4012474133.md)接口同步积分。

#### Q：如何开启积分补录电子围栏？

A：可发邮件开启&关闭积分补录围栏，详细参考：

邮件标题：XXX商圈确认开启&关闭手动提交地理围栏限制

邮件内容：商圈商户号、商圈名称、说明开启或关闭围栏

收件人：zhsq@tencent.com

请商圈商户注意确认：已知悉商圈快速积分手动提交地理围栏限制开启后，消费者只能在商圈内进行手动提交积分信息。

#### Q：什么是商圈单店？

A：商圈单店指无圈外交易的单店，即该收款商户号只有当前门店在使用。由积分引擎自动标记，无法人工干预。

#### Q：微信圈店时一个门店最多可以圈多少个商户号？

A：一个门店最多关联10个收款商户号。

#### Q：支付完成后上传位置信息，能否匹配商圈并触发自动积分及推送？

A：不能。支付后上传位置无法匹配商圈自动积分，但当天订单可通过手动积分方式补录。

#### Q：商圈报错：（错误码 2729564220），什么原因？

A：openid解析失败，可以参考[获取openid指引](https://pay.weixin.qq.com/doc/v3/partner/4012081935.md#OpenID)重新获取openid和传入的openid对比。

#### Q： [商圈会员快速积分插件](https://mp.weixin.qq.com/wxopen/plugindevdoc?appid=wxfab2bf944bfc4da6&token=1213946737&lang=zh_CN) member\_status传1和传0，有什么区别？

A：传1表示已开通会员卡，传0表示未开通会员卡。

#### Q：自动积分条件是什么？

A：自动积分条件：

1、停车到场(需开发，调用[商圈会员停车状态同步](https://pay.weixin.qq.com/doc/v3/partner/4012474127.md)接口，通过同步用户停车进场/离场状态，关联其在商圈内消费行为，实现自动积分)；

2、门店下单有回传在所属商圈的门店编号(需连锁商家配合微信侧进行相关操作，持续性工作目前暂时无法快速解决)；

3、支付前5小时内定位命中商圈边界(需持续让用户有打开小程序的认知)；

4、无圈外交易的单店。

#### Q：商圈不支持香港钱包支付的订单积分，是指商户号还是用户不支持？

A：商圈不支持用户使用非中国大陆银行卡支付的订单积分。

#### Q：商圈积分是支付前上传位置，还是支付后？

A：支付前调用wx.getLocation()上传位置。

#### Q：如何导出已圈店的所有店铺商户号?

A：请参考[圈店工具使用文档](https://docs.qq.com/doc/DYllDRXZFYld2TW12)中的第八点。

#### Q：门店收款商户号上的标签有什么含义

本次商圈圈店升级放开了门店商户号必须在商圈边界内有交易的限制。

为了防止实际操作圈店的执行人错误操作导致圈入实际不属于商圈业态的门店商户号，造成积分损失，我们将一些特殊的商户号打上标识，供商圈管理方查看管理。各标签及含义如下：

|     |     |
| --- | --- |
| 标签 | 含义 |
| 在商圈内无线下交易 | 未识别到商户号在商圈边界内有线下交易，多出现在扫码点餐、新店等情况，圈店人确认后仍可圈入 |
| 有大量商圈外交易 | 识别到商户号在商圈外也有大量交易。对于商圈的影响如下，可点击确定即可圈店。<br>1） 配置营销券，可能会在场外核销；<br>2） 积分，可能会造成用户会在非商圈的地方发生消费，手动提交获取积分 |
| 有商圈外交易 | 同上 |
| 有少量商圈外交易 | 同上 |
| 线上商家 | 判断商户号有线上交易行为，部分扫码点餐、或门店小程序支付商户号会被打上此标签。圈店人确认后仍可圈入 |
| 出行商家 | 判断商户号为出行类商家，如嘀嘀打车等。圈店人确认后仍可圈入 |
| 民生商家 | 判断商户号为民生类商家，如手机充值、电信缴费等。圈店人确认后仍可圈入 |

#### Q：​ 服务商模式商圈想变更为直连模式是否支持？

A：服务商模式转直连模式：支持商圈商户号是服务商模式因业务调整变更为直连模式，参考以下指引发邮件申请：

邮件标题：商圈服务商配置申请修改为xx（直连）-xx（商圈名称）-xx（日期）

邮件内容：

商圈名称：

品牌主商户号：

原商圈服务商商户号：

新商圈商户号：

修改原因：

承诺：接受变更后，原商圈服务商商户号的圈店、接口调用权限全部失效。商圈自行做好切换过渡期的准备工作。

#### Q：同一个card\_id下，openid和code是否为一对一关系？

A：openid与code没有关系。openid是用户在appid下的唯一标识，code是用户在商圈会员卡card\_id下的唯一标识。
