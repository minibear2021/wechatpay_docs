## 一、整体开发流程概览

注意事项：

开发联调阶段，调用接口后，建议在日志中保留应答的HTTP头 `Request-ID` 值， `Request-ID` 作为[请求的唯一标识](https://pay.weixin.qq.com/doc/v3/partner/4012081726.md#%E8%AF%B7%E6%B1%82%E7%9A%84%E5%94%AF%E4%B8%80%E6%A0%87%E8%AF%86)，在调用接口遇到问题时，可向微信侧提供该值用于快速定位到请求记录，协助排查问题原因。

### 1、【必需接入】摇优惠接入

“摇一摇有优惠”是微信支付推出的一种互动营销功能，基于用户在微信支付的消费偏好，向用户提供最短路径优惠「发放」（支付完成页下发优惠券）和「使用」优惠的能力，从而帮助商户实现交易闭环与增长。商家需完成商品券接入、投放计划接入、投放计划测试、上线投放计划这4个步骤才能正式上线摇优惠活动。

步骤说明

| 步骤 | 步骤名称 | 操作方式 | 步骤说明 |
| :-- | :-- | :-- | :-- |
| 1 | 摇优惠商品券接入 | 服务商调用API接口 | 商品券是微信支付为商户提供的电子优惠券全链路解决方案，支持服务商通过API接口实现优惠券的创建、领取、通知、核销，券信息查询等配套功能完成商品券的管理操作。详细指引请参考：[商品券开发指引](https://pay.weixin.qq.com/doc/v3/partner/4015788446.md)。 |
| 2 | 投放计划 API 接入 | 服务商调用投放计划API 接口 | 投放计划是商家投放端的统一创建和管理能力，支持服务商通过 API 接口实现投放计划的创建、管理、查询、状态变更通知等完整能力。 |
| 3 | 投放计划测试 | 方式一：服务商联系微信支付运营配置 | / |
| 方式二：品牌方登录经营平台配置 | 完成商品券功能开发后，品牌方可登录[品牌经营平台](https://pay.weixin.qq.com/xdc/brandhomeweb/brand/home#/)创建“投放计划”，创建成功后会由微信侧进行人工审核，一般需要2个工作日完成审核。<br>为确保投放计划正常运行，需在审核期间提前进行的测试。通过测试帮助开发者和运营人员在上线前完成系统联调和功能验证。<br>创建投放计划、测试等流程请参考：[品牌经营平台管理投放计划](https://pay.weixin.qq.com/doc/v3/partner/4016111064.md)。<br>注意：若测试时长不足，请在审核通过前及时联系微信运营延后审核。 |
| 方式三：服务商通过投放计划 API配置 | 服务商调用投放计划 API 进行活动配置 |
| 4 | 上线投放计划 | / | 完成测试后，等待审核通过，投放计划将正式上线。 |

### 2、【按需接入】核销组件接入

为解决在摇一摇有优惠场景“由于用户在商家小程序未登录正确的领券账号，导致无法找到优惠券”和“用户领取优惠券后在商家小程序未能找到优惠券适用的商品”这两大核心问题，平台侧提供标准化核销组件，用于提升用户领券率与领取核销率。

- 核销组件接入指引请参考：[核销组件开发指引](https://pay.weixin.qq.com/doc/v3/partner/4016110741.md)。

- 完成核销组件开发后，需对投放计划重新进行测试，两种方式：

  方式一：服务商联系微信支付运营配置；

  方式二：品牌方登录[品牌经营平台](https://pay.weixin.qq.com/xdc/brandhomeweb/brand/home#/)配置，详细指引请参考：[品牌经营平台管理投放计划](https://pay.weixin.qq.com/doc/v3/partner/4016111064.md)。


注意

目前从微信卡包页面点击微信卡券的“立即使用”跳转不会携带[核销组件的参数](https://pay.weixin.qq.com/doc/v3/partner/4016110741.md#step2%E3%80%81%E5%88%A4%E6%96%AD%E7%94%A8%E6%88%B7%E7%99%BB%E5%BD%95%E8%B4%A6%E5%8F%B7%E7%8A%B6%E6%80%81)，商户只有接入核销组件且从摇一摇会场“去小程序使用”跳转才会携带核销组件的参数，商户侧需要做兼容处理。

### 3、【按需接入】品牌门店接入

品牌录入门店后可以在开展摇一摇有优惠活动时，基于门店位置更精准地投放商品券，提升领券转化率，从而提升品牌和活动可拿到的曝光量。

品牌方案，是微信支付为「连锁经营品牌商家」定制的行业解决方案。方案基于“品牌-门店”的商业关系，向连锁经营品牌商家提供进件、交易场景配置、流量开放、资金管理、数据管理、营销管理等场景的专项解决方案。

服务商可以通过API接口管理品牌门店：API详见：[品牌门店API](https://pay.weixin.qq.com/doc/v3/partner/4015783183.md)。

注意

接入品牌后，面向交易用户的收银台、支付完成页、支付凭证、交易账单将优先展示品牌名称和品牌logo，接入前请详细了解[面向交易用户展示品牌信息的相关规则](https://pay.weixin.qq.com/doc/v3/partner/4016110939.md)。
