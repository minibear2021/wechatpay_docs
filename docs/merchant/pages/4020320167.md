## 简介

商家转账用户授权免确认收款模式下，商户通过在移动端应用APP中集成开放SDK调起微信请求用户授权免确认收款。

## 接入前注意事项

在接入前需详细阅读下方说明：

商户需先通过[发起免确认收款授权](https://pay.weixin.qq.com/doc/v3/merchant/4015901167.md)接口申请免确认收款授权，获取到跳转页面的package信息后，商户APP再通过微信[Open SDK](https://developers.weixin.qq.com/doc/oplatform/Mobile_App/WeChat_Pay/Vendor_Service_Center.html)（详见[Android](https://developers.weixin.qq.com/doc/oplatform/Mobile_App/Access_Guide/Android.html)接入指南）的sendReq方法拉起用户免确认收款授权页面。

## 接口说明

支持商户：【普通商户】

接口名称：WXOpenBusinessView

需要引用新的openSDK：

- Android openSDK下载地址（版本>=5.3.1）：[Android资源下载](https://developers.weixin.qq.com/doc/oplatform/Downloads/Android_Resource.html)

- Android 接入文档链接：[openSDK说明文档](https://developers.weixin.qq.com/doc/oplatform/Mobile_App/Access_Guide/Android.html)


接口兼容：

Android兼容性表现：若微信版本>=8.0.45.51，开发者可以通过此openSDK接口调起用户免确认收款授权页面；若微信版本< 8.0.45.51，开发者通过此openSDK接口不能跳转到微信或跳转微信后无法拉起用户免确认收款授权页面，此时开发者应在拉起微信前提示用户更新微信版本。

## 接口参数

Android对应对象：WXOpenBusinessView.Req

## 请求参数

| 参数名 | 类型 | 必填 | 描述 |
| --- |  --- | --- | --- |
| businessType | string(16) | true | 【业务类型】固定配置：requestMerchantTransfer |
| query | string(2048) | true | 【查询参数】使用URL的query string方式传递参数，格式为key=value&key2=value2，其中value、value2需要进行UrlEncode处理。 |
| query.mchId | string(32) | true | 【商户号】商户号，由微信支付生成并下发，和发起授权申请传入的mchid必须是同一个 |
| query.appId | string(32) | true | 【商户AppID】微信开放平台审核通过的移动应用appid，和发起授权申请传入的appid必须是同一个 |
| query.package | string(1024) | true | 【package信息】对应[发起免确认收款授权](https://pay.weixin.qq.com/doc/v3/merchant/4015901167.md)接口应答参数中的package\_info（仅当单据状态为WAIT\_USER\_CONFIRM: 待收款用户确认时才返回），用于唤起用户免确认收款授权页面。 |

调用示例

Android示例

```
int wxSdkVersion = api.getWXAppSupportAPI();
if (wxSdkVersion >= 0x28002d33) {
  WXOpenBusinessView.Req req = new WXOpenBusinessView.Req();
  req.businessType = "requestMerchantTransfer";
  req.query = "mchId=1230000000&appId=wx8888888888888888&package=0002-affffddafdfafddffda%3D%3D";
  Boolean ret = api.sendReq(req);
} else {
  /*需提示用户升级微信版本*/
}
```
## 返回参数

Android对应对象：WXOpenBusinessView.Resp

| 参数名 | 类型 | 必填 | 描述 |
| --- |  --- | --- | --- |
| businessType | string(16) | true | 【业务类型】打开的业务类型。 |
| extMsg | string | true | 【扩展信息】返回的业务数据，格式为JSON字符串，如 `{"result":"success"}`。具体内部字段如下 |
| extMsg.result | string | true | 【结果信息】<br>`success`：展示页面成功。<br>`fail`：展示页面失败。<br>`cancel`：用户取消。发生场景：用户未确认授权，点击取消，返回APP |
