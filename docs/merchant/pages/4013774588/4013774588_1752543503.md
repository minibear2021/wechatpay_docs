## 1. 概述

当商户发起转账场景为现金营销时，需通过发起转账接口中传入以下参数。其中，transfer\_scene\_id参数需传入对应场景ID；user\_recv\_perception参数可选择传入对应的用户收款感知；此外，还需通过transfer\_scene\_report\_infos参数传入转账场景报备信息，用于报备该笔转账资金的转账背景。

transfer\_scene\_report\_infos为数组类型参数，在现金营销的转账场景下需固定传两条明细，每条“转账场景报备信息明细”包含info\_type、info\_content两个参数。

## 2.转账场景ID传值要求

需在发起转账接口通过transfer\_scene\_id【转账场景ID】参数传入对应的场景ID，此参数值为固定值，请传入：1000

## 3.用户收款感知传值要求

需在发起转账接口通过user\_recv\_perception【用户收款感知】参数传入对应的收款感知，当前仅支持传入【现金奖励】或【活动奖励】，不传则默认展示【活动奖励】。

## 4.转账场景报备信息参数传值要求

需在发起转账接口通过transfer\_scene\_report\_infos【转账场景报备信息】参数传入对应的报备信息，以报备该笔转账资金的转账背景。

transfer\_scene\_report\_infos参数传值要求：

|     |     |     |     |     |     |     |
| --- | --- | --- | --- | --- | --- | --- |
| 字段简称 | 字段名 | 类型\[长度限制\] | 是否必填 | 示例值 | 描述 | 传值示例 |
| 信息类型 | info\_type | string \[1, 15\] | 必填 | 活动名称 | 信息类型，两条明细中必须分别填写以下两个取值：<br>- 活动名称<br>  <br>- 奖励说明<br>  <br>注：信息类型不可重复填写，请严格按照示例传入 | "transfer\_scene\_report\_infos" :  \[{<br>    "info\_type" :   "活动名称",<br>    "info\_content" : "新会员有礼"<br>},{<br>    "info\_type" : "奖励说明",<br>    "info\_content" : "注册会员抽奖一等奖"<br>}\] |
| 信息内容 | info\_content | string \[1, 32\] | 必填 | 新会员有礼 | 信息内容，商户自定义内容，如注册会员抽奖一等奖 |
