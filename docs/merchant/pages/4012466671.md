使用场景：商户创建活动后，可以通过该接口增加支付有礼的发券商户号，用于管理活动。

可调用商户：商户/服务商/渠道商




## 接口说明

支持商户：【普通商户】

请求方式：【POST】 `/v3/marketing/paygiftactivity/activities/{activity_id}/merchants/add`

请求域名：【主域名】https://api.mch.weixin.qq.com 使用该域名将访问就近的接入点

　　　　　【备域名】https://api2.mch.weixin.qq.com 使用该域名将访问异地的接入点，指引[点击查看](https://pay.weixin.qq.com/doc/v3/merchant/4012075113.md)

## 请求参数

### Header 参数
| 参数名 | 类型 | 必填 | 描述 |
| --- |  --- | --- | --- |
| Authorization | string | true | 请参考 [签名认证](https://pay.weixin.qq.com/doc/v3/merchant/4012365342.md#%E8%AF%B7%E6%B1%82%E6%8E%A5%E5%8F%A3%E5%90%8E%E7%AB%AF%E7%AD%BE%E5%90%8D) 生成认证信息 |
| Accept | string | true | 请设置为 `application/json` |
| Content-Type | string | true | 请设置为 `application/json` |
### Path 参数
| 参数名 | 类型 | 必填 | 描述 |
| --- |  --- | --- | --- |
| activity_id | string(20) | true | **【活动Id】** 活动Id |
### Body 参数
| 参数名 | 类型 | 必填 | 描述 |
| --- |  --- | --- | --- |
| merchant_id_list | array[string(15)] | true | **【添加的发券商户号】** 新增到活动中的发券商户号列表 |
| add_request_no | string(128) | true | **【请求业务单据号】** 商户添加发券商户号的凭据号，商户侧需保持唯一性 |
## 应答参数

200 OK

| 参数名 | 类型 | 必填 | 描述 |
| --- |  --- | --- | --- |
| activity_id | string | false | **【活动Id】** 活动Id |
| invalid_merchant_id_list | array[object] | false | **【校验失败的发券商户号】** 未通过规则校验的发券商户号列表 |
| invalid_merchant_id_list.mchid | string(15) | false | **【商户Id】** 商户Id |
| invalid_merchant_id_list.invalid_reason | string | false | **【无效原因】** 活动参与商户校验失败的原因 |
| add_time | string | false | **【添加时间】** 成功添加发券商户号的时间 |
