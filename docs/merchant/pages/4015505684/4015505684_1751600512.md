当调用[查询分账结果接口](https://pay.weixin.qq.com/doc/v3/merchant/4012525210.md)返回字段 `receivers.fail_reason` 分账失败原因时，对应的处理方案可参考以下内容：

| fail\_reason字段返回的分账失败原因 | 商户处理方案 |
| :-- | :-- |
| ACCOUNT\_ABNORMAL：分账接收账户异常 | 1、如果是分账给个人，商户侧需引导分账用户检查自己的微信是否被管控，如果被管控需要微信支付公众号下发的管控及解脱消息处理，商户核实用户解除管控后，再次发起分账请求。若用户无法解脱或者未看到管控消息，引导用户联系微信支付客服，告知用户微信号无法收款，请客服协助。待客服处理完成后，商户再次发起分账请求。<br>2、如果是分账给商户，商户需引导接收方商户申诉处理管控，可参考：[商户自查违约记录及申诉方法(移动端)](https://kf.qq.com/faq/240129aQZjiM240129QRBFRV.html)或[商户自查违约记录及申诉方法(PC端)](https://kf.qq.com/faq/240129iuEZBJ240129meAR3u.html)申诉解除管控，商户核实接收方解除管控成功后，再发起分账请求。 |
| NO\_RELATION：分账关系已解除 | 商户需先调用接口[添加分账接收方](https://pay.weixin.qq.com/doc/v3/merchant/4012528995.md)后，再发起分账请求。 |
| RECEIVER\_HIGH\_RISK：高风险接收方 | 风控原因，该用户/商户无法分账，后续也无需再次发起分账请求，商户需自行通过其他方式处理。 |
| RECEIVER\_REAL\_NAME\_NOT\_VERIFIED：接收方未实名 | 1、商户引导分账接收用户进行实名，微信支付会通过公众号下发的实名验证消息，用户按照消息进行处理，再发起分账接口请求；<br>2、若用户已经实名，还是该报错，引导用户联系微信支付客服，告知用户微信号无法收款，请客服协助。 |
| NO\_AUTH：分账权限已解除 | 分账分出方商户无权限，需先申请分账权限，再发起分账。 |
| RECEIVER\_RECEIPT\_LIMIT：超出用户月收款限额 | 收款用户限额，商户和与用户协商，下个月再发起分账请求进行分账，或者可引导用户开通微信支付商户号，从个人微信收款切换到商户号收款，重新添加分账接收方后，再发起分账请求。 |
| PAYER\_ACCOUNT\_ABNORMAL：分出方账户异常 | 需引导分账分出方商户申诉处理管控，可参考：[商户自查违约记录及申诉方法(移动端)](https://kf.qq.com/faq/240129aQZjiM240129QRBFRV.html)或[商户自查违约记录及申诉方法(PC端)](https://kf.qq.com/faq/240129iuEZBJ240129meAR3u.html)申诉解除管控，待分出方申诉解除管控成功后，再发起分账请求。 |
| INVALID\_REQUEST：描述参数设置失败 | 分账描述包含敏感词，请修改【分账描述】字段后，重新发起分账请求。 |
