发起分账请求后，可调用此接口查询分账结果

1. 发起解冻剩余资金请求后，可调用此接口查询解冻剩余资金的结果





## 接口说明

支持商户：【普通商户】

请求方式：【GET】 `/v3/profitsharing/orders/{out_order_no}`

请求域名：【主域名】https://api.mch.weixin.qq.com 使用该域名将访问就近的接入点

　　　　　【备域名】https://api2.mch.weixin.qq.com 使用该域名将访问异地的接入点，指引[点击查看](https://pay.weixin.qq.com/doc/v3/merchant/4012075113.md)

## 请求参数

### Header 参数
| 参数名 | 类型 | 必填 | 描述 |
| --- |  --- | --- | --- |
| Authorization | string | true | 请参考 [签名认证](https://pay.weixin.qq.com/doc/v3/merchant/4012365342.md#%E8%AF%B7%E6%B1%82%E6%8E%A5%E5%8F%A3%E5%90%8E%E7%AB%AF%E7%AD%BE%E5%90%8D) 生成认证信息 |
| Accept | string | true | 请设置为 `application/json` |
### Path 参数
| 参数名 | 类型 | 必填 | 描述 |
| --- |  --- | --- | --- |
| out_order_no | string(64) | true | 【商户分账单号】 申请分账时的商户分账单号 |
### Query 参数
| 参数名 | 类型 | 必填 | 描述 |
| --- |  --- | --- | --- |
| transaction_id | string(32) | true | 【微信订单号】 微信支付订单号 |
## 应答参数

200 OK

| 参数名 | 类型 | 必填 | 描述 |

| receivers | array[object] | true | 【分账接收方列表】 分账接收方列表 |

| receivers.account | string(64) | true | 【接收方账号】<br><ul><li>分账接收方类型为MERCHANT\_ID时，分账接收方账号为商户号</li><li>分账接收方类型为PERSONAL\_OPENID时，分账接收方账号为个人OpenID（由商户的AppID转换得到）</li></ul> |

| receivers.fail_reason | string | false | 【分账失败原因】 分账失败原因，当分账结果result为CLOSED（已关闭）时，返回该字段，具体处理方案请参考 [分账失败处理指引](https://pay.weixin.qq.com/doc/v3/merchant/4015505684.md)。<br>可选取值<br><ul><li>`ACCOUNT_ABNORMAL`:  分账接收账户异常</li><li>`NO_RELATION`:  分账关系已解除</li><li>`RECEIVER_HIGH_RISK`:  高风险接收方</li><li>`RECEIVER_REAL_NAME_NOT_VERIFIED`:  接收方未实名</li><li>`NO_AUTH`:  分账权限已解除</li><li>`RECEIVER_RECEIPT_LIMIT`:  超出用户月收款限额</li><li>`PAYER_ACCOUNT_ABNORMAL`:  分出方账户异常</li><li>`INVALID_REQUEST`:  描述参数设置失败</li></ul> |
| receivers.create_time | string | true | 【分账创建时间】 分账创建时间，需遵循 [RFC3339](https://datatracker.ietf.org/doc/html/rfc3339) 标准格式：yyyy-MM-DDTHH:mm:ss+TIMEZONE。yyyy-MM-DD 表示年月日；T 字符用于分隔日期和时间部分；HH:mm:ss 表示具体的时分秒；TIMEZONE 表示时区（例如，+08:00 对应东八区时间，即北京时间）。示例：2015-05-20T13:29:35+08:00 表示北京时间2015年5月20日13点29分35秒。 |
| receivers.finish_time | string | true | 【分账完成时间】 分账完成时间，需遵循 [RFC3339](https://datatracker.ietf.org/doc/html/rfc3339) 标准格式：yyyy-MM-DDTHH:mm:ss+TIMEZONE。yyyy-MM-DD 表示年月日；T 字符用于分隔日期和时间部分；HH:mm:ss 表示具体的时分秒；TIMEZONE 表示时区（例如，+08:00 对应东八区时间，即北京时间）。示例：2015-05-20T13:29:35+08:00 表示北京时间2015年5月20日13点29分35秒。 |
| receivers.detail_id | string(64) | true | 【分账明细单号】 微信分账明细单号，每笔分账业务执行的明细单号，可与资金账单对账使用，对应资金账单中的“业务凭证号” |
