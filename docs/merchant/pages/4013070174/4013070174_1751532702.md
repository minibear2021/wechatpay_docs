## 1、已有商户号申请APP支付产品权限

若商户已有商户号(未开通APP支付权限)，需新增APP支付权限，可通过【[商户平台](https://pay.wechatpay.cn/index.php/core/home/login) -> 产品中心 -> APP支付 -> 申请开通 -> 填写APPID、APP页面截图及APP在应用市场的上架链接 -> 提交申请】操作，待审核(7个工作日内)通过后将开通APP支付权限。

| 步骤一：登录商户平台，点击“产品中心” | 步骤二：点击“APP支付” |
| :-- | :-- |

| 步骤三：点击“申请开通” | 步骤四：请依次填写APPID、上传APP页面截图，并提供上架到主流应用市场上的APP下载页面的链接。<br>仔细阅读相关协议，确认无误后，勾选“我已阅读”，随后点击“提交申请”。 |
| ![](https://gtimg.wechatpay.cn/resource/xres/mmpaydoc/static/img/f53eb538cd22c9e567bbb492c7061e96.png) | ![](https://gtimg.wechatpay.cn/resource/xres/mmpaydoc/static/img/6672b222048e745a1a074ac8a3e0ef5d.png) |
| 填写说明 |
| APPID | 1、商户如未申请移动应用的appid，可参考以下指引申请：第一步[注册开放平台账号](https://developers.weixin.qq.com/doc/oplatform/Third-party_Platforms/2.0/operation/open/create.html)，第二步[创建移动应用APPID](https://developers.weixin.qq.com/doc/oplatform/Mobile_App/guideline/create.html)。<br>创建移动应用APPID时，若为Android系统或鸿蒙系统的应用，填写的应用包名、应用签名（在鸿蒙系统中对应为Bundle ID和Identifier）需与商户App项目配置文件中的信息一致，后续调起APP支付时会进行校验，不一致将会无法调起支付。<br>2、商户如已申请移动应用的appid，请参考指引[查看APPID](https://pay.weixin.qq.com/doc/v3/merchant/4013289251.md#2.1.3%E3%80%81%E7%A7%BB%E5%8A%A8%E5%BA%94%E7%94%A8)。<br>注意：已有商户号申请权限填写的appid，不会自动发起商户号与该APPID的授权绑定，需要商户手动操作绑定，具体请参考：[商户号绑定APPID账号操作指南](https://pay.weixin.qq.com/doc/v3/merchant/4013289251.md#2.2%E3%80%81%E7%99%BB%E5%BD%95%E5%95%86%E6%88%B7%E5%B9%B3%E5%8F%B0%E5%8F%91%E8%B5%B7APPID%E7%9A%84%E6%8E%88%E6%9D%83%E7%BB%91%E5%AE%9A%E7%94%B3%E8%AF%B7)。 |
| APP页面截图 | 提供APP上架到应用市场的截图、APP首页截图、APP内售卖的商品或者服务展示页截图和APP尾页截图（尾页可以用软著替代）。请参考[《申请接入微信支付的商户App截图标准》](https://kf.qq.com/faq/161226bymQ3y161226yQbmQv.html) |
| APP下载链接 | APP必须已经上架到应用市场，请填入上架到主流应用市场(如App Store，应用宝)上的APP下载页面的链接。<br>注：下载页面需包含应用名称、应用图标、应用介绍、开发商信息、下载按钮等基本信息。 |

## 2、申请新商户号和APP支付产品权限

若商户没有商户号，请参考[微信支付商户入驻详细流程](https://kf.qq.com/faq/180910eUfeyu180910BFZNJN.html)指引申请商户号([入驻支持的主体类型](https://kf.qq.com/faq/180910IBZVnQ180910naQ77b.html))，当在“填写经营与行业信息”这一步骤时，必需选择APP经营场景(其他场景可按需选择)，同时填写移动应用APPID(因为在使用APP支付时，商户号必须与开发平台注册的移动应用APPID建立绑定关系，才能正常支付)和APP页面截图。

按照流程指引入驻成功后，商户号将默认开通开通APP支付产品权限。

| 申请过程中需特别注意的步骤 | 字段填写说明 |
| :-- | :-- |
| ![](https://gtimg.wechatpay.cn/resource/xres/mmpaydoc/static/img/b70b23fc8918cdfd4a8ebe6e465045ae.png) | 商户简称 | 1、 不支持单纯以人名来命名，若为个体户经营，可用“个体户+经营者名称”或“经营者名称+业务”命名，如“个体户张三”或“张三餐饮店”；<br>2、不支持无实际意义的文案，如“XX特约商户”、“800”、“XX客服电话XXX”；<br>3、填写的商户简称会在交易场景中显示，参考：[入驻填写的商户简称展示方式](https://kf.qq.com/faq/161226RbUVvU161226JjY3M7.html) |
| 客户电话 | 1、请填写真实有效的客服电话，将在交易记录中向买家展示，提供咨询服务。<br>2、请确保电话畅通，以便入驻后微信侧回拨确认。 |
| 所属行业 | 所属行业请参考：[入驻结算规则、行业属性及特殊资质](https://kf.qq.com/faq/220228IJb2UV220228uEjU3Q.html)。 |
| 应用APPID | 1、商户如未申请移动应用的appid，可按照步骤申请：第一步[注册开放平台账号](https://developers.weixin.qq.com/doc/oplatform/Third-party_Platforms/2.0/operation/open/create.html)，第二步[创建移动应用APPID](https://developers.weixin.qq.com/doc/oplatform/Mobile_App/guideline/create.html)。<br>创建移动应用APPID时，若为Android系统或鸿蒙系统的应用，填写的应用包名、应用签名（在鸿蒙系统中对应为Bundle ID和Identifier）需与商户App项目配置文件中的信息一致，后续调起APP支付时会进行校验，不一致将会无法调起支付。<br>2、商户如已申请移动应用的appid，请参考指引[查看APPID](https://pay.weixin.qq.com/doc/v3/merchant/4013289251.md#2.1.3%E3%80%81%E7%A7%BB%E5%8A%A8%E5%BA%94%E7%94%A8)。<br>注意：入驻时填写的appid，微信系统会自动发起商户号与该APPID的授权绑定，商户参考指引[确认授权绑定商户号](https://pay.weixin.qq.com/doc/v3/merchant/4013289251.md#2.3.3%E3%80%81%E7%A7%BB%E5%8A%A8%E5%BA%94%E7%94%A8)。 |
| APP页面截图 | 提供APP首页、尾页、应用内和支付页截图(不需要实现功能)，截图标准请参考[《申请接入微信支付的商户App截图标准》](https://kf.qq.com/faq/161226bymQ3y161226yQbmQv.html) |
| APP下载链接 | 若APP已经上架到应用市场，商户可选填上架到主流应用市场(如App Store，应用宝)上的APP下载页面的链接。<br>注：下载页面需包含应用名称、应用图标、应用介绍、开发商信息、下载按钮等基本信息。 |
