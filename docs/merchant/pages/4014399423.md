根据商户单号查询用户免确认收款的授权结果。

一、通过[发起免确认收款授权](https://pay.weixin.qq.com/doc/v3/merchant/4015901167.md) 获取授权的查询规则

- 对于用户超期未确认而关闭的授权记录，系统设有 30天​ 的数据保留期

  - 保留期内查询：接口会正常返回授权详情，其中授权状态为 CLOSED ，关闭原因为 USER\_OVERDUE\_UNCONFIRMED

  - 超期后查询：接口将返回 NOT\_FOUND 错误

二、使用[发起转账并完成免确认收款授权](https://pay.weixin.qq.com/doc/v3/merchant/4014399293.md)获取授权的查询规则

- 若用户收款成功且免确认收款授权成功

  - 接口返回的HTTP状态码为200，且授权状态为 TAKING\_EFFECT
- 若用户未收款成功或收款成功但未同意免确认收款授权，或者因业务规则、风控策略等限制无法完成授权

  - 系统将不会处理授权信息，接口将返回NOT\_FOUND 错误

注意：

- 用户授权成功后，该授权长期有效，直至用户或商户系统主动解除授权。 商户系统需保存商户侧授权单号out\_authorization\_no 和 微信免确认收款授权单号authorization\_id ，避免无法再发起转账

- 单个商户的接口频率限制为100次/s





## 接口说明

支持商户：【普通商户】

请求方式：【GET】 `/v3/fund-app/mch-transfer/user-confirm-authorization/out-authorization-no/{out_authorization_no}`

请求域名：【主域名】https://api.mch.weixin.qq.com 使用该域名将访问就近的接入点

　　　　　【备域名】https://api2.mch.weixin.qq.com 使用该域名将访问异地的接入点，指引[点击查看](https://pay.weixin.qq.com/doc/v3/merchant/4012075113.md)

| 状态码 | 错误码 | 描述 | 解决方案 |
| --- | --- | --- | --- |
| 400 | PARAM\_ERROR | 参数错误 | 请根据错误提示正确传入参数 |
| 400 | INVALID\_REQUEST | HTTP 请求不符合微信支付 APIv3 接口规则 | 请参阅 [接口规则](https://pay.weixin.qq.com/doc/v3/merchant/4012081709.md) |
| 401 | SIGN\_ERROR | 验证不通过 | 请参阅 [签名常见问题](https://pay.weixin.qq.com/doc/v3/merchant/4012072670.md) |

| 404 | NOT\_FOUND | 免确认收款授权不存在 | 检查传入的授权单号 |
| 429 | RATELIMIT\_EXCEEDED | 频率超限 | 请降低请求接口频率后，使用相同参数重试 |
