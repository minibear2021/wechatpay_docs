## 1. 概述

当商户发起转账场景为二手回收时，需通过发起转账接口中传入以下参数。其中，transfer\_scene\_id参数需传入对应场景ID；user\_recv\_perception参数可选择传入对应的用户收款感知；此外，还需通过transfer\_scene\_report\_infos参数传入转账场景报备信息，用于报备该笔转账资金的转账背景。

transfer\_scene\_report\_infos为数组类型参数，在二手回收的转账场景下需固定传一条明细，该条“转账场景报备信息明细”包含info\_type、info\_content两个参数。

## 2.转账场景ID传值要求

需在发起转账接口通过transfer\_scene\_id【转账场景ID】参数传入对应的场景ID，此参数值为固定值，请传入：1010

## 3.用户收款感知传值要求

需在发起转账接口通过user\_recv\_perception【用户收款感知】参数传入对应的收款感知，当前仅支持传入【二手回收货款】；若未传入该参数，默认展示为【二手回收货款】。

## 4.转账场景报备信息参数传值要求

需在发起转账接口通过transfer\_scene\_report\_infos【转账场景报备信息】参数传入对应的报备信息，以报备该笔转账资金的转账背景。

transfer\_scene\_report\_infos参数传值要求：

|     |     |     |     |     |     |     |
| --- | --- | --- | --- | --- | --- | --- |
| 字段简称 | 字段名 | 类型\[长度限制\] | 是否必填 | 示例值 | 描述 | 传值示例 |
| 信息类型 | info\_type | string \[1, 15\] | 必填 | 回收商品名称 | 信息类型，只可填写以下取值：<br>- 回收商品名称 | "transfer\_scene\_report\_infos" :  \[{<br>    "info\_type" :   "回收商品名称",<br>    "info\_content" : "塑料瓶"<br>}\] |
| 信息内容 | info\_content | string \[1, 32\] | 必填 | 塑料瓶 | 信息内容，商户自定义内容，如塑料瓶 |
