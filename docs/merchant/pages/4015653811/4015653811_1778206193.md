根据商户单号解除免确认收款授权。

注：单个商户的接口频率限制为50次/s




## 接口说明

支持商户：【普通商户】

请求方式：【POST】 `/v3/fund-app/mch-transfer/user-confirm-authorization/out-authorization-no/{out_authorization_no}/close`

请求域名：【主域名】https://api.mch.weixin.qq.com 使用该域名将访问就近的接入点

　　　　　【备域名】https://api2.mch.weixin.qq.com 使用该域名将访问异地的接入点，指引[点击查看](https://pay.weixin.qq.com/doc/v3/merchant/4012075113.md)

## 请求参数

### Header 参数
| 参数名 | 类型 | 必填 | 描述 |
| --- |  --- | --- | --- |
| Authorization | string | true | 请参考 [签名认证](https://pay.weixin.qq.com/doc/v3/merchant/4012365342.md#%E8%AF%B7%E6%B1%82%E6%8E%A5%E5%8F%A3%E5%90%8E%E7%AB%AF%E7%AD%BE%E5%90%8D) 生成认证信息 |
| Accept | string | true | 请设置为 `application/json` |
### Path 参数
| 参数名 | 类型 | 必填 | 描述 |
| --- |  --- | --- | --- |
| out_authorization_no | string(32) | true | 【商户侧授权单号】 商户系统内部的免确认收款授权单号。 |
## 应答参数

200 OK

| 参数名 | 类型 | 必填 | 描述 |
| --- |  --- | --- | --- |
| out_authorization_no | string(32) | true | 【商户侧授权单号】 商户系统内部的免确认收款授权单号。 |
| appid | string(32) | true | 【商户AppID】 商户应用唯一标识 |
| openid | string(64) | true | 【收款用户OpenID】 用户在商户appid下的唯一标识 |
| user_display_name | string(32) | true | 【用户展示昵称】 用户免确认收款授权详情中展示的昵称 |
| authorization_id | string(32) | false | 【微信免确认收款授权单号】 用户确认授权后微信支付返回的授权单号 |
| state | string | true | 【授权状态】 授权状态<br>可选取值<br><ul><li>`CLOSED`:  用户或商户系统关闭，无法转账至该授权对应的用户</li></ul> |
| authorize_time | string(32) | false | 【用户确认授权的时间】 用户确认授权后返回。按照使用rfc3339所定义的格式，格式为yyyy-MM-DDThh:mm:ss+TIMEZONE |
| close_info | object | false | 【免确认收款授权的关闭信息】 当状态为已关闭时返回，如关闭原因、关闭时间等 |
| close_info.close_time | string(32) | true | 【关闭时间】 当状态为已关闭时返回，按照使用rfc3339所定义的格式，格式为yyyy-MM-DDThh:mm:ss+TIMEZONE |
| close_info.close_reason | string | true | 【免确认收款授权的原因】 当状态为已关闭时返回<br>可选取值<br><ul><li>`CLOSE_VIA_MCH_API`:  商户通过API主动关闭授权</li></ul> |
| transfer_scene_id | string(36) | false | 【转账场景ID】 用户授权后免确认收款的转账场景，可前往“商户平台-产品中心-商家转账”中申请。如：1000（现金营销），1006（企业报销）等 |
| user_recv_perception | string(256) | false | 【用户收款感知】 商户申请免确认收款授权时传入的内容 |
| create_time | string | false | 【单据创建时间】 商户申请授权的时间，按照使用rfc3339所定义的格式，格式为yyyy-MM-DDThh:mm:ss+TIMEZONE |
## 错误码

以下是本接口返回的错误码列表。详细错误码规则，请参考[微信支付接口规则-错误码和错误提示](https://pay.weixin.qq.com/doc/v3/merchant/4012081709.md#%E9%94%99%E8%AF%AF%E7%A0%81%E5%92%8C%E9%94%99%E8%AF%AF%E6%8F%90%E7%A4%BA)

| 状态码 | 错误码 | 描述 | 解决方案 |
| --- | --- | --- | --- |
| 400 | PARAM\_ERROR | 参数错误 | 请根据错误提示正确传入参数 |
| 400 | INVALID\_REQUEST | HTTP 请求不符合微信支付 APIv3 接口规则 | 请参阅 [接口规则](https://pay.weixin.qq.com/doc/v3/merchant/4012081709.md) |
| 401 | SIGN\_ERROR | 验证不通过 | 请参阅 [签名常见问题](https://pay.weixin.qq.com/doc/v3/merchant/4012072670.md) |
| 500 | SYSTEM\_ERROR | 系统异常，请稍后重试 | 请稍后重试 |
| 404 | NOT\_FOUND | 免确认收款授权不存在 | 检查传入的授权单号 |
| 429 | RATELIMIT\_EXCEEDED | 频率超限 | 请降低请求接口频率后，使用相同参数重试 |
