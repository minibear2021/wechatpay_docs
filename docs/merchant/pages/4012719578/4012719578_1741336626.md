## 简介

商家转账用户确认模式下，商户通过在移动端应用APP中集成开放SDK调起微信请求用户确认收款。

## 接入前注意事项

在接入前需详细阅读下方说明：

商户需先通过[发起转账](https://pay.weixin.qq.com/doc/v3/merchant/4012716434.md#%E5%BA%94%E7%AD%94%E5%8F%82%E6%95%B0)接口申请创建转账单，获取到跳转领取页面的package信息后，商户APP再通过微信[Open SDK](https://developers.weixin.qq.com/doc/oplatform/Mobile_App/WeChat_Pay/Vendor_Service_Center.html)（详见[IOS](https://developers.weixin.qq.com/doc/oplatform/Mobile_App/Access_Guide/iOS.html)接入指南）的sendReq方法拉起用户确认收款页。

## 接口说明

支持商户：【普通商户】

接口名称：WXOpenBusinessView

需要引用新的openSDK：

- iOS openSDK下载地址（版本>=1.8.4）：[iOS资源下载](https://developers.weixin.qq.com/doc/oplatform/Downloads/iOS_Resource.html)

- iOS 接入文档链接：[openSDK说明文档](https://developers.weixin.qq.com/doc/oplatform/Mobile_App/Access_Guide/iOS.html)


接口兼容：

iOS兼容性表现：若微信版本 >= 8.0.45，开发者可以通过此openSDK接口调起用户确认收款页面；若微信版本 < 8.0.45，开发者通过此openSDK接口可以跳转到微信，但不能调起用户确认收款页面，此时微信会提示用户可能由于应用的请求非法或者微信版本过低。

## 接口参数

iOS对应对象：WXOpenBusinessViewReq

## 请求参数

| 参数名 | 类型 | 必填 | 描述 |
| --- |  --- | --- | --- |
| businessType | string(16) | true | 【业务类型】固定配置：requestMerchantTransfer |
| query | string(2048) | true | 【查询参数】使用URL的query string方式传递参数，格式为key=value&key2=value2，其中value、value2需要进行UrlEncode处理。 |
| query.mchId | string(32) | true | 【商户号】商户号，由微信支付生成并下发，和发起转账传入的mchid必须是同一个 |
| query.appId | string(32) | true | 【商户AppID】微信开放平台审核通过的移动应用appid，和发起转账传入的appid必须是同一个 |
| query.package | string(1024) | true | 【package信息】对应[发起转账](https://pay.weixin.qq.com/doc/v3/merchant/4012716434.md#%E5%BA%94%E7%AD%94%E5%8F%82%E6%95%B0)接口应答参数中的 package\_info（仅当转账单据状态为WAIT\_USER\_CONFIRM: 待收款用户确认时才返回），用于唤起用户确认收款页面。 |

调用示例

iOS示例

```
WXOpenBusinessViewReq *req = [WXOpenBusinessViewReq object];
req.businessType = @"requestMerchantTransfer";
req.query = @"mchId=1230000000&appId=wx8888888888888888&package=affffddafdfafddffda%3D%3D";
[WXApi sendReq:req]
```

## 返回参数

iOS对应对象：WXOpenBusinessViewResp

| 参数名 | 类型 | 必填 | 描述 |
| --- |  --- | --- | --- |
| businessType | string(16) | true | 【业务类型】打开的业务类型。 |
| extMsg | string | true | 【扩展信息】返回的业务数据，格式为JSON字符串，如 {"result":"success"}。具体内部字段如下 |
| extMsg.result | string | true | 【结果信息】<br>success: 展示页面成功。<br>fail: 展示页面失败。<br>cancel: 用户取消。发生场景：用户未确认收款，点击取消，返回APP |

返回示例

```
// 展示成功的返回数据示例
resp.businessType == "requestMerchantTransfer"
resp.extMsg == "{\"result\": \"success\"}"
```

注意：

带有返回结果信息不代表订单确认成功，具体状态需以接口查询的结果为准。
