## 1 什么是商家转账

商家转账目前仅支持微信商户向用户微信零钱转账，为商户提供免费、安全的转账服务。支持集成到商家自有业务系统，需有研发能力可接入。商户需在用户收款流程中，拉起微信官方页面，由用户确认收款方式后方可成功转账，资金实时到账，转账成功的资金不支持退回。

（1）用户收款体验

流程示意如下，仅绿框内页面为微信官方页面，需按照商家转账规则展示，其余流程均为商户灵活设计。

1. 商户LOGO，展示发起转账的AppID的LOGO，修改操作路径如下：

   2. 公众号的appid需登录[微信公众平台](https://mp.weixin.qq.com/)-设置与开发-账号设置，点击头像进入修改入口进行修改

（2）产品特色

- 提升转账效率、安全性

  - 本产品支持用户点击收款后即时入账。且提供了免费的安全医生服务、系统漏洞诊断，有严格的安全策略实时保护体系，24小时保障商户资金安全。提供更快、更安全的收款体验。
- 用户明确收款方式，降低商家运营成本

  - 用户主动选择入账微信，避免用户对账不清引发客诉。

  - 用户微信账户存在异常需要处理时，可闭环在微信内解脱、完成收款，减少商家客服介入。

## 2 商户使用流程

使用商家转账到零钱的流程包括：商户申请开通商家转账、充值资金、发起转账、查询转账记录和获取转账账单及电子回单。

### 2.1 开通商家转账

- 申请权限：按照[权限申请](https://pay.weixin.qq.com/doc/v3/merchant/4013740645.md)开通产品权限

- 接入产品能力：按照[开发指引](https://pay.weixin.qq.com/doc/v3/merchant/4012715211.md)进行系统开发


### 2.2 充值资金

- [充值步骤说明](https://pay.weixin.qq.com/doc/v3/merchant/4013747269.md)


### 2.3 发起转账

请根据接口文档对接，相关产品规则如下：

#### （1）配置ip地址

- [设置接口安全IP说明](https://pay.weixin.qq.com/doc/v3/merchant/4013751010.md)


#### （2）按转账场景发起转账

- 在发起转账时，需按实际情况传入转账场景ID（你可在[商户平台](https://pay.wechatpay.cn/index.php/core/home/login?return_url=%2F)-产品中心-商家转账-产品设置获取及查看场景ID）。

- 当前已支持的场景可参考下表，用户收款时感知 的收款原因将根据转账场景自动展示默认内容，如下表中备注为默认展示的部分。

- 如有其他展示需求，可在发起转账时接口主动传入对应的“用户收款感知”，可支持传入的内容见下表：


| 转账场景 | 场景介绍 | 用户收款感知 | 备注 |
| :-- | :-- | :-- | :-- |
| 现金营销 | 向参与营销活动的用户发放现金奖励 | 活动奖励 | 默认展示 |
| 现金奖励 | 需在发起转账时，“用户收款感知”字段主动传入“现金奖励”才可展示 |
| 企业赔付 | 交易后商家向用户进行赔偿或退款 | 退款 | 默认展示 |
| 商家赔付 | 需在发起转账时，“用户收款感知”字段主动传入“商家赔付”才可展示 |
| 佣金报酬 | 向劳务提供方，如销售、团长、主播支付佣金、报酬等 | 劳务报酬 | 默认展示 |
| 报销款 | 需在发起转账时，“用户收款感知”字段主动传入“报销款”才可展示 |
| 企业补贴 | 需在发起转账时，“用户收款感知”字段主动传入“企业补贴”才可展示 |
| 开工利是 | 需在发起转账时，“用户收款感知”字段主动传入“开工利是”才可展示 |
| 采购货款 | 向个人供应商支付货款、采购款、服务费等 | 货款 | 默认展示 |
| 二手回收 | 向回收物品的用户发放货款 | 二手回收货款 | 默认展示 |
| 公益补助 | 公益机构向受助用户群体发放补助 | 公益补助金 | 默认展示 |
| 行政补贴 | 行政机构向用户发放补贴、奖励金 | 行政补贴 | 默认展示 |
| 行政奖励 | 需在发起转账时，“用户收款感知”字段主动传入“行政奖励”才可展示 |
| 保险理赔 | 保险机构向用户发放保险金 | 保险理赔款 | 默认展示 |

- 以佣金报酬场景为例，默认展示为劳务报酬（图1），若发起转账时在“用户收款感知”传入“开工利是”，则展示为开工利是（图2）。

#### （3）按转账场景报备背景信息

- 此外，需在发起转账接口传入对应的 报备信息： 包括“信息类型”、“信息内容”，以报备该笔转账资金的转账背景。

- 各场景需传入的信息类型见下表， 有多个字段时需填写完整，如：转账场景为现金营销时，需填入活动名称、奖励说明。


| 转账场景 | 信息类型 | 信息内容填写说明 |
| :-- | :-- | :-- |
| 现金营销 | 活动名称 | 请在信息内容描述用户参与活动的名称，如新会员有礼 |
| 奖励说明 | 请在信息内容描述用户因为什么奖励获取这笔资金，如注册会员抽奖一等奖 |
| 企业赔付 | 赔付原因 | 请在信息内容描述用户接收当前这笔赔付或退款的原因，如商品质量问题退款 |
| 佣金报酬 | 岗位类型 | 请在信息内容描述收款用户的岗位类型，如外卖员、专家顾问 |
| 报酬说明 | 请在信息内容描述用户接收当前这笔报酬的原因，如7月份配送费，高温补贴 |
| 采购货款 | 采购商品名称 | 请在信息内容描述向个人用户采购的商品名称 |
| 二手回收 | 回收商品名称 | 请在信息内容描述用户回收的商品名称 |
| 公益补助 | 公益活动名称 | 请在信息内容描述公益活动在民政部的备案名称 |
| 公益活动备案编号 | 请在信息内容描述公益活动在民政部的备案编号 |
| 行政补贴 | 补贴类型 | 请在信息内容描述补贴类型，如购车补贴、见义勇为奖励 |
| 保险理赔 | 保险产品备案编号 | 请在信息内容描述保险产品备案编号 |
| 保险名称 | 请在信息内容描述保险产品名称 |
| 保险操作单号 | 请在信息内容描述当前这笔转账资金的操作单号 |

#### （4）支持实名校验

- 商家转账支持校验收款方的真实姓名来进一步确认收款人，需在发起转账时，传入用户姓名，若校验与当前收款用户的微信实名不一致将不会执行转账。

- 当单笔转账金额在0.3元（含）至2,000元（不含）之间时，商户可以自主决定是否传入用户姓名。当单笔转账金额低于0.3元时，不支持传入用户姓名校验。

- 当单笔转账金额>=2,000元时，为保障商户出资金安全，必须传入用户姓名进行校验。


### 2.4 查询转账记录及申请相关凭证

- 查询转账记录：可根据[商户单号查询转账单](https://pay.weixin.qq.com/doc/v3/merchant/4012716437.md)、[微信转账单号查询转账单](https://pay.weixin.qq.com/doc/v3/merchant/4012716457.md)接口文档对接。当前API仅支持查询30天内的转账单。

- 获取转账账单：可通过[商户平台](https://pay.wechatpay.cn/index.php/core/home/login?return_url=%2F)-交易中心-资金账单，查询和下载账单。次日10点后可下载前一天账单（仅包含已完成的转账记录）。

- 获取电子回单：可根据[商户单号查询电子回单](https://pay.weixin.qq.com/doc/v3/merchant/4012716436.md)、[微信单号查询电子回单](https://pay.weixin.qq.com/doc/v3/merchant/4012716455.md)接口文档对接。商家转账支持商户申请电子回单（通过实名校验且转账成功的转账单，在6个月以内），用于证明转账行为的真实性，便于审计和报税。电子回单格式可参考下图
