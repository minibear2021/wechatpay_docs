商户可以通过该接口查看回单申请进度

| 回单状态 | 含义 | 建议处理方式 |
| --- | --- | --- |
| GENERATING | 表示电子回单正在生成中 | 过段时间重试查询，建议查询时间间隔不低于2分钟 |
| FINISHED | 回单已成功生成，会返回回单文件的下载地址和摘要信息 | 通过返回的下载地址下载回单文件数据并比较摘要值 |
| FAILED | 电子回单生成失败，会返回具体的失败原因 | 停止重试查询，查看具体的失败原因 |

请务必对比下载的回单文件的摘要值与查询接口返回的摘要值的一致性，确保得到的回单文件的真实性和完整性。

下载地址的有效期为10分钟，超过10分钟后需要重新通过该接口获取下载地址（不需要重新申请）。




## 接口说明

支持商户：【普通商户】

请求方式：【GET】 `/v3/fund-app/mch-transfer/elecsign/transfer-bill-no/{transfer_bill_no}`

请求域名：【主域名】https://api.mch.weixin.qq.com 使用该域名将访问就近的接入点

　　　　　【备域名】https://api2.mch.weixin.qq.com 使用该域名将访问异地的接入点，指引[点击查看](https://pay.weixin.qq.com/doc/v3/merchant/4012075113.md)

## 应答参数

200 OK

| 参数名 | 类型 | 必填 | 描述 |
| --- |  --- | --- | --- |
| state | string | true | 【申请单状态】 GENERATING：生成中，FINISHED: 已完成，FAILED：已失败<br>可选取值<br><ul><li>`GENERATING`:  表示当前电子回单已受理成功并在处理中</li><li>`FINISHED`:  表示当前电子回单已处理完成</li><li>`FAILED`:  表示当前电子回单生成失败，失败原因字段会返回具体的失败原因</li></ul> |
| create_time | string(32) | true | 【申请单创建时间】 申请单的创建时间，按照使用rfc3339所定义的格式，格式为yyyy-MM-DDThh:mm:ss+TIMEZONE |
| update_time | string(32) | true | 【申请单更新时间】 申请单最近的更新时间，按照使用rfc3339所定义的格式，格式为yyyy-MM-DDThh:mm:ss+TIMEZONE |
| hash_type | string | false | 【回单文件摘要类型】 申请单状态为已完成时返回<br>可选取值<br><ul><li>`SHA256`:  SHA256摘要算法</li><li>`SM3`:  国密SM3摘要算法</li></ul> |
| hash_value | string(512) | false | 【回单文件摘要值】 申请单状态为已完成时返回，请务必将下载的回单文件摘要值与该值进行对比，以保证回单文件的完整性和真实性 |
| download_url | string(3000) | false | 【回单文件下载地址】 申请单状态为已完成时返回，有效期10分钟，过期后可以重新通过该接口获取；下载地址的域名，路径，参数都可能变化，请不要自行拼接 |
